**Root cause:** SchoolYearProvider's `useEffect` fires `loadSchoolYears()` whenever `isAuthenticated` becomes `true`. For super admin, this calls `GET /api/school-year` which goes through `authenticateToken` middleware that looks up the teacher collection -- super admins are not teachers, so it returns 401.

**Fix:** Import `useAuth` and get the `user` object. Check `user?.isSuperAdmin` before loading school years. Use `user` (not `localStorage.getItem('loginType')`) to avoid the race condition identified in Pitfall 1 of the research -- `user` is set in React state simultaneously with `isAuthenticated`, while localStorage might not be written yet.

**Step 1:** The file already imports `useAuth` at line 9. The provider already destructures `isAuthenticated` from it at line 22. Change line 22 to also destructure `user`:
```jsx
const { isAuthenticated, user } = useAuth()
```

**Step 2:** Modify the `useEffect` (lines 29-39) to skip loading for super admin:
```jsx
useEffect(() => {
  if (isAuthenticated) {
    // Super admin has no school years -- skip tenant-scoped loading
    if (user?.isSuperAdmin) {
      setCurrentSchoolYear(null)
      setSchoolYears([])
      setIsLoading(false)
      setError(null)
      return
    }
    loadSchoolYears()
  } else {
    // Clear data when not authenticated
    setCurrentSchoolYear(null)
    setSchoolYears([])
    setIsLoading(false)
    setError(null)
  }
}, [isAuthenticated, user?.isSuperAdmin])
```

**Important:** Add `user?.isSuperAdmin` to the dependency array so if the user object changes (e.g., from super admin to regular via logout/login), the effect re-runs correctly.

**Do NOT** modify anything else in this file. The rest of the SchoolYearProvider (create, update, delete, helpers) should remain unchanged -- super admin will never call them.
