Add a batch endpoint that returns per-member attendance rates for an orchestra, enabling the frontend to show smart suggestions (likely present / frequent absentee indicators).

Purpose: AUX-02 requires the frontend to know each member's historical attendance rate before the conductor starts marking. A single batch endpoint avoids N+1 requests for individual student stats.

Output: New GET /orchestra/:orchestraId/member-attendance-rates endpoint returning an array of { studentId, totalRehearsals, attended, late, attendanceRate, suggestion } objects.
</objective>

  <files>api/orchestra/orchestra.service.js</files>
  <action>
Add a new exported function `getMemberAttendanceRates(orchestraId, options = {})` to orchestra.service.js.

Implementation:
1. Extract tenantId via requireTenantId(options.context?.tenantId)
2. Load the orchestra document to get its memberIds array
3. Query activity_attendance collection with an aggregation pipeline:
   - Match: `{ groupId: orchestraId, tenantId, isArchived: { $ne: true } }`
   - Group by studentId: count total records, count present records using MINISTRY_PRESENT_STATUSES, count late records (status === 'איחור')
4. Build result array mapping each memberId to its stats:
   - For members with attendance records: `{ studentId, totalRehearsals, attended, late, attendanceRate: (attended/total)*100 }`
   - For members with zero records: `{ studentId, totalRehearsals: 0, attended: 0, late: 0, attendanceRate: 0 }`
5. Add suggestion field to each entry:
   - `attendanceRate > 80` => suggestion: 'likelyPresent'
   - `attendanceRate < 50 && totalRehearsals >= 3` => suggestion: 'frequentAbsent'
   - Otherwise => suggestion: null
   - The threshold of 3 minimum rehearsals prevents flagging new members as absentees

Use MINISTRY_PRESENT_STATUSES from config/constants.js (already imported in the file).
Use the MongoDB $cond aggregation operator for counting present/late within the $group stage.

Export the function in the module.exports at the bottom of the file.
