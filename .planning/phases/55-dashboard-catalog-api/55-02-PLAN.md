<task type="auto">
  <name>Task 1: Add getCatalog to registry that groups reports into 4 categories</name>
  <files>api/reports/report.registry.js</files>
  <action>
Add a new exported function `getCatalog(userRoles)` to `report.registry.js`.

**Category mapping** (5 generator categories -> 4 catalog categories):
- `teacher` -> `teacher` (label: `מורים`, icon: `Users`)
- `student` -> `student` (label: `תלמידים`, icon: `GraduationCap`)
- `institutional` -> `institutional` (label: `מוסדי`, icon: `Building`)
- `department` + `schedule` -> `department-schedule` (label: `מחלקות ולו"ז`, icon: `Grid`)

**Category definitions** (define as a const array at module level):
```js
const CATALOG_CATEGORIES = [
  { id: 'teacher', label: 'מורים', icon: 'Users', generatorCategories: ['teacher'] },
  { id: 'student', label: 'תלמידים', icon: 'GraduationCap', generatorCategories: ['student'] },
  { id: 'institutional', label: 'מוסדי', icon: 'Building', generatorCategories: ['institutional'] },
  { id: 'department-schedule', label: 'מחלקות ולו"ז', icon: 'Grid', generatorCategories: ['department', 'schedule'] },
];
```

**getCatalog logic:**
1. Call the existing `getRegistry(userRoles)` to get the flat role-filtered list.
2. For each CATALOG_CATEGORIES entry, filter reports where `report.category` is in `generatorCategories`.
3. Return array of `{ id, label, icon, reports: [...] }`. Only include categories that have at least one report after filtering.

Keep the existing `getRegistry` function unchanged (backward compatible). `getCatalog` is additive.
  </action>
  <verify>
`grep -n 'getCatalog\|CATALOG_CATEGORIES' api/reports/report.registry.js` shows both defined. `node -e "import('./api/reports/report.registry.js').then(m => console.log(typeof m.getCatalog))"` prints `function`.
  </verify>
  <done>
`getCatalog(userRoles)` returns an array of 4 category objects, each with `id`, `label`, `icon`, and `reports` array. Categories with no visible reports are omitted.
  </done>
</task>

This preserves backward compatibility while adding the grouped structure.
