  <action>
In `bulkCreateRehearsals`, AFTER date generation (`generateDatesForDayOfWeek`) and BEFORE the rehearsal document creation loop, add a conflict pre-check that validates ALL generated dates:

1. Import `checkRehearsalConflicts` is already available from Plan 01.

2. After `const utcDates = generateDatesForDayOfWeek(...)`, iterate over all dates and check each for conflicts:
   ```js
   // Pre-validate all dates for conflicts before inserting any
   const dateConflicts = [];
   for (const utcDate of utcDates) {
     const conflicts = await checkRehearsalConflicts(
       { date: utcDate, startTime, endTime, location, groupId: orchestraId },
       { context: options.context }
     );
     if (conflicts.hasConflicts) {
       dateConflicts.push({
         date: formatDate(utcDate, 'YYYY-MM-DD'),
         roomConflicts: conflicts.roomConflicts,
         teacherConflicts: conflicts.teacherConflicts,
       });
     }
   }

   if (dateConflicts.length > 0) {
     const err = new Error('Scheduling conflicts detected for bulk creation');
     err.code = 'BULK_CONFLICT';
     err.dateConflicts = dateConflicts;
     err.totalDates = utcDates.length;
     err.conflictingDates = dateConflicts.length;
     throw err;
   }
   ```

3. This is an all-or-nothing check: if ANY date has a conflict, zero rehearsals are inserted. The user can then add conflicting dates to `excludeDates` and retry.

4. Performance note: For a typical bulk creation (30-40 dates in a school year), this runs 30-40 individual conflict checks. Each check runs 6 parallel queries. This is acceptable because it happens before any writes and prevents much worse cleanup scenarios. Add a log line: `console.log('Validating ${utcDates.length} dates for conflicts...')`.

  <action>
In the `bulkCreateRehearsals` controller function, update the catch block to handle the `BULK_CONFLICT` error code. Currently it has a generic 500 catch-all. Add before the 500 response:

```js
if (err.code === 'BULK_CONFLICT') {
  return res.status(409).json({
    error: 'Scheduling conflicts detected',
    message: `${err.conflictingDates} out of ${err.totalDates} dates have conflicts`,
    dateConflicts: err.dateConflicts,
    totalDates: err.totalDates,
    conflictingDates: err.conflictingDates,
  });
}
```

Also handle the single-conflict case (code `CONFLICT`) in case `bulkCreateRehearsals` ever throws it:
```js
if (err.code === 'CONFLICT') {
  return res.status(409).json({
    error: 'Scheduling conflict detected',
    conflicts: err.conflicts,
  });
}
```

The response format for bulk conflicts gives the frontend enough information to:
- Show which specific dates have conflicts
- Show what each conflict is (room double-booking vs teacher double-booking)
- Suggest adding those dates to `excludeDates`
