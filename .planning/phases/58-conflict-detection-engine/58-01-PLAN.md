<task type="auto">
  <name>Task 1: Create rehearsalConflictService.js with cross-source room and teacher conflict detection</name>
  <files>services/rehearsalConflictService.js</files>
  <action>
Create `services/rehearsalConflictService.js` exporting a single function `checkRehearsalConflicts(rehearsalData, options)`.

The function accepts:
- `rehearsalData` object: `{ date, startTime, endTime, location, groupId }` (date is a Date or string, times are HH:MM)
- `options`: `{ context: { tenantId, schoolYearId }, excludeRehearsalId }` (excludeRehearsalId used when updating)

Returns: `{ hasConflicts, roomConflicts, teacherConflicts }` where each conflict object includes: `{ type, activityType, activityName, conflictingTime, room, conflictId }`.

Implementation steps:

1. **Room conflict detection** - Query all 3 activity sources for the same date + room + overlapping time:
   - **Rehearsals**: Query `rehearsal` collection for `{ tenantId, location, date: same-day-range }`, exclude self via `excludeRehearsalId`. Use `doTimesOverlap()` from `utils/timeUtils.js` to filter.
   - **Theory lessons**: Query `theory_lesson` collection for `{ tenantId, location, date: same-day-range, isActive: true }`. Use `doTimesOverlap()`.
   - **Time blocks**: Query `teacher` collection for teachers with `teaching.timeBlocks` matching the same `location` and overlapping time on the same day of week. Use the rehearsal's `dayOfWeek` (derived from date via `getDayOfWeek`). TimeBlocks don't have dates -- they repeat weekly, so match by `dayOfWeek` and location.

2. **Teacher/conductor conflict detection** - Look up the orchestra's `conductorId` from the `groupId`, then query:
   - **Other rehearsals**: Find rehearsals on the same date with a `groupId` pointing to an orchestra where `conductorId` matches this orchestra's conductor. Exclude self.
   - **Theory lessons**: Find theory lessons on the same date where `teacherId === conductorId` and times overlap.
   - **Time blocks**: Find time blocks for the conductor teacher on the same dayOfWeek with overlapping times.

3. **Conflict response format** - Each conflict object:
   ```js
   {
     type: 'room' | 'teacher',
     activityType: 'חזרה' | 'תאוריה' | 'שיעור פרטי',
     activityName: string,  // orchestra name, theory category, or 'שיעור פרטי'
     conflictingTime: '10:00-11:30',
     room: string,
     conflictId: string,
     description: string  // Hebrew: "החדר X תפוס..." or "המנצח/ת תפוס/ה..."
   }
   ```

4. Run all 3 room queries and all 3 teacher queries in parallel using `Promise.all` for performance.

5. Import from existing utils: `doTimesOverlap` from `utils/timeUtils.js`, `getCollection` from `services/mongoDB.service.js`, `requireTenantId` from `middleware/tenant.middleware.js`, `createAppDate`/`getStartOfDay`/`getEndOfDay`/`getDayOfWeek` from `utils/dateHelpers.js`.

Do NOT modify the existing `conflictDetectionService.js` (theory-specific, leave it alone).

**rehearsal.service.js changes:**

1. Import `checkRehearsalConflicts` from `../../services/rehearsalConflictService.js`.

2. In `addRehearsal`, AFTER validation and date conversion but BEFORE the `withTransaction` block, call:
   ```js
   const conflicts = await checkRehearsalConflicts(
     { date: value.date, startTime: value.startTime, endTime: value.endTime, location: value.location, groupId: value.groupId },
     { context: options.context }
   );
   if (conflicts.hasConflicts) {
     const err = new Error('Scheduling conflict detected');
     err.code = 'CONFLICT';
     err.conflicts = conflicts;
     throw err;
   }
   ```

3. In `updateRehearsal`, AFTER validation and date conversion but BEFORE the `findOneAndUpdate` call, add conflict check. Only check if time/location/date fields are being changed. Fetch the existing rehearsal first to merge unchanged fields:
   ```js
   const existing = await getRehearsalById(rehearsalId, options);
   const mergedData = {
     date: value.date || existing.date,
     startTime: value.startTime || existing.startTime,
     endTime: value.endTime || existing.endTime,
     location: value.location || existing.location,
     groupId: value.groupId || existing.groupId,
   };
   const conflicts = await checkRehearsalConflicts(
     mergedData,
     { context: options.context, excludeRehearsalId: rehearsalId }
   );
   if (conflicts.hasConflicts) {
     const err = new Error('Scheduling conflict detected');
     err.code = 'CONFLICT';
     err.conflicts = conflicts;
     throw err;
   }
   ```

   ```js
       error: 'Scheduling conflict detected',
       conflicts: err.conflicts,
     });
   }
   ```
