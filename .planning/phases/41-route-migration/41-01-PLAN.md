<objective>
Expand DEFAULT_ROLE_PERMISSIONS to match existing requireAuth access patterns, fix role normalization in buildContext, then migrate 7 core-domain route files from requireAuth(roles[]) to requirePermission(domain, action).

Purpose: The permission matrix created in Phase 39 is missing several domain entries that teaching-tier roles currently access via requireAuth. Without fixing these first, migrating to requirePermission would cause 403 regressions. This task closes those gaps, adds role normalization, and migrates the 7 core route files -- fulfilling PERM-06 for core teaching domains.

Output: Updated permission matrix + buildContext with role normalization + 7 route files using requirePermission instead of requireAuth, with identical access patterns for existing roles.
</objective>
