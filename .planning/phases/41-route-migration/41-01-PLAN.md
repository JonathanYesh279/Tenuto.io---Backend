<task type="auto">
  <name>Task 1: Add legacy role normalization in buildContext</name>
  <files>middleware/tenant.middleware.js</files>
  <action>
In `middleware/tenant.middleware.js`, fix a runtime bug where teachers with old role names ('מורה תאוריה', 'מנצח', 'מגמה') get empty effectivePermissions because those names are not in DEFAULT_ROLE_PERMISSIONS.

1. Add `ROLE_RENAME_MAP` to the existing import from `'../config/constants.js'`:
   ```js
   import { ADMIN_TIER_ROLES, COORDINATOR_ROLES, ROLE_RENAME_MAP } from '../config/constants.js';
   ```

2. In the `buildContext` function, BEFORE the line `const teacherRoles = teacher.roles || [];`, add role normalization:
   ```js
   // Normalize legacy role names (e.g. 'מורה תאוריה' -> 'תאוריה') for permission resolution.
   // Does NOT mutate the teacher document -- only normalizes for this request's context.
   const rawRoles = teacher.roles || [];
   const teacherRoles = rawRoles.map(r => ROLE_RENAME_MAP[r] || r);
   ```
   Remove the old `const teacherRoles = teacher.roles || [];` line -- the new code replaces it.

3. This ensures `resolveEffectivePermissions(teacherRoles, rolePermissions)` on the next line will find the correct permission entries for legacy role names.

IMPORTANT: Do NOT mutate `teacher.roles` itself -- other middleware and controllers may still check for the original role names. Only the `teacherRoles` local variable used for context should be normalized.

Also ensure `req.context.userRoles` uses the NORMALIZED roles (teacherRoles), which is already the case since the existing code passes `teacherRoles` to `req.context.userRoles`.
  </action>
  <verify>
Run: `grep "ROLE_RENAME_MAP" middleware/tenant.middleware.js` -- must show import and usage.
Run: `node -e "import('./middleware/tenant.middleware.js').then(() => console.log('OK')).catch(e => console.error(e))"` -- must print OK.
Verify the normalization logic: `grep -A2 "Normalize legacy" middleware/tenant.middleware.js` -- must show the map call.
  </verify>
  <done>buildContext normalizes legacy role names before resolving permissions; teachers with 'מורה תאוריה' will resolve to 'תאוריה' permissions; teacher.roles is not mutated.</done>
