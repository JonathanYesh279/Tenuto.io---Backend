**auth.route.js** -- CRITICAL: /api/auth is mounted WITHOUT buildContext in server.js:

The /api/auth route is mounted as `app.use('/api/auth', authRoutes)` -- no buildContext middleware in the chain. requirePermission needs `req.context.effectivePermissions` which is populated by buildContext. Without it, requirePermission returns 500 PERMISSION_CONTEXT_MISSING.

Fix: Import buildContext from tenant.middleware.js and chain it between authenticateToken and requirePermission for each admin route.

1. Change the import to:
   ```js
   import { authenticateToken, requirePermission } from '../../middleware/auth.middleware.js'
   import { buildContext } from '../../middleware/tenant.middleware.js'
   ```

2. For each admin route, chain: `authenticateToken, buildContext, requirePermission(...)`:
   - POST /migrate-users -> `authenticateToken, buildContext, requirePermission('settings', 'update')`
   - POST /migrate-invitations -> `authenticateToken, buildContext, requirePermission('settings', 'update')`
   - GET /invitation-stats -> `authenticateToken, buildContext, requirePermission('settings', 'view')`
   - GET /check-teacher/:email -> `authenticateToken, buildContext, requirePermission('teachers', 'view')`
   - DELETE /remove-teacher/:email -> `authenticateToken, buildContext, requirePermission('teachers', 'delete')`

The public routes (login, refresh, forgot-password, etc.) and protected routes (validate, logout, change-password) do NOT use requireAuth -- leave them unchanged.
