```js
export const DEFAULT_ATTENDANCE_ALERT_SETTINGS = {
  consecutiveAbsences: 3,

  minimumRehearsals: 3,
  isEnabled: true
};
```
- `consecutiveAbsences`: number of consecutive absences before flagging (default 3)

- `minimumRehearsals`: minimum rehearsals before rate-based flagging applies (reuses Phase 60 decision of 3 minimum)
- `isEnabled`: whether attendance alerting is active for this tenant

<task type="auto">
  <name>Task 2: Create attendance alert service with flagging, dashboard, and student summary</name>
  <files>api/attendance-alerts/attendanceAlert.service.js</files>
  <action>
Create `api/attendance-alerts/attendanceAlert.service.js` with these exported functions:

**1. `getTenantAlertSettings(tenantId)`**
- Read tenant doc, return `tenant.settings?.attendanceAlerts || DEFAULT_ATTENDANCE_ALERT_SETTINGS`
- Import DEFAULT_ATTENDANCE_ALERT_SETTINGS from config/constants.js

**2. `evaluateFlaggedStudents(orchestraId, options)`**
- `options` has `context` with tenantId
- Get orchestra doc to retrieve memberIds
- Get alert settings via getTenantAlertSettings
- If not enabled, return empty array
- For each member, query activity_attendance (isArchived: {$ne: true}, tenantId) ordered by date desc
- Check two flag conditions:
  a. **Consecutive absences**: Walk records from most recent. Count consecutive records where status is NOT in MINISTRY_PRESENT_STATUSES. If count >= settings.consecutiveAbsences, flag with reason `consecutive_absences` and count
  b. **Absence rate**: If total records >= settings.minimumRehearsals, calculate absence rate. If absence percentage >= settings.absenceRateThreshold, flag with reason `high_absence_rate` and rate value
- Return array: `[{ studentId, studentName, flags: [{ reason, value, threshold }], attendanceRate, totalRehearsals, consecutiveAbsences }]`
- Only return students that have at least one flag
- For student names: batch-fetch student docs with projection `personalInfo.firstName, personalInfo.lastName`
- Use MINISTRY_PRESENT_STATUSES from config/constants.js (consistent with Phase 59/60 decisions)

**3. `getAttendanceDashboard(options)`**
- `options` has `context` with tenantId, optional `startDate`, `endDate`
- Returns:
  a. **perOrchestra**: For each active orchestra in tenant, aggregate activity_attendance to get: orchestraId, orchestraName, conductorName, totalRehearsals, averageAttendanceRate, flaggedStudentCount
  b. **monthlyTrends**: Aggregate activity_attendance by month (last 6 months), return [{month: "YYYY-MM", totalSessions, attendanceRate}]
  c. **flaggedStudents**: Call evaluateFlaggedStudents for each orchestra, flatten and deduplicate by studentId (student may be in multiple orchestras), include orchestraName on each flag entry
  d. **summary**: { totalOrchestras, totalStudentsTracked, overallAttendanceRate, totalFlagged }
- For perOrchestra aggregation: use MongoDB aggregation on activity_attendance grouped by activityId (rehearsal), then compute per-orchestra stats
- Orchestras: query orchestra collection with tenantId + isActive:true
- Conductors: batch-fetch teacher names for conductor display

**4. `getStudentAttendanceSummary(studentId, options)`**
- `options` has `context` with tenantId
- Query activity_attendance for this student (isArchived: {$ne: true}, tenantId), sorted by date desc
- Calculate: totalSessions, attendedCount, lateCount, absentCount, attendanceRate (using MINISTRY_PRESENT_STATUSES)
- Get last 10 sessions as recentHistory: [{date, activityType, activityName, status}]
- Compute consecutiveAbsences (walk from most recent)
- Get tenant alert settings, evaluate flags for this student
- Return: { studentId, attendanceRate, totalSessions, attended, late, absent, consecutiveAbsences, recentHistory, flags, alertSettings: { consecutiveAbsences: threshold, absenceRateThreshold } }

All functions must use `requireTenantId(options.context?.tenantId)` pattern. Use `createLogger('attendanceAlert.service')` for logging. Import getCollection from services/mongoDB.service.js, ObjectId from mongodb.
  </action>

  <done>Service exports evaluateFlaggedStudents, getAttendanceDashboard, getStudentAttendanceSummary, getTenantAlertSettings. All functions use tenant-scoped queries and MINISTRY_PRESENT_STATUSES for consistency.</done>

**2. Create `api/attendance-alerts/attendanceAlert.route.js`:**

```js
import express from 'express';
import { attendanceAlertController } from './attendanceAlert.controller.js';
import { requirePermission } from '../../middleware/auth.middleware.js';

const router = express.Router();

// Tenant alert settings
router.get('/settings', requirePermission('settings', 'view'), attendanceAlertController.getAlertSettings);

// Flagged students for a specific orchestra
router.get('/orchestra/:orchestraId/flagged', requirePermission('orchestras', 'view'), attendanceAlertController.getFlaggedStudents);

// Admin attendance dashboard
router.get('/dashboard', requirePermission('reports', 'view'), attendanceAlertController.getDashboard);

// Student attendance summary
router.get('/student/:studentId/summary', requirePermission('students', 'view'), attendanceAlertController.getStudentSummary);

export default router;
```
