Add auto-save, attendance history dots, and collapsible notes field.

1. **Auto-save with debounce:** Add `hasInteracted` ref (starts false, set true on first status change). Add `saveTimerRef` using useRef. After any status change in `updateAttendance`, if `hasInteracted` is true, clear previous timer and set new 1500ms timeout to call `saveAttendance()`. Add a save indicator state: `'idle' | 'saving' | 'saved'`. Show in the sticky header:
   - `idle`: nothing
   - `saving`: small spinner + "שומר..." in gray
   - `saved`: small green CheckCircle + "נשמר" that fades after 2 seconds
   Remove the manual "שמור נוכחות" button. Auto-save replaces it. Keep the `alert()` calls replaced with the subtle indicator. On save error, show a red inline error "שגיאה בשמירה" in the sticky header that allows retry.

2. **History dots (last 3 sessions):** After loading rehearsal attendance, also fetch the student's recent attendance from the attendance-alerts API. For each member, call `apiService.attendanceAlerts.getStudentSummary(member.id)` to get `recentHistory`. Since calling this per-student is expensive, batch it: after members load, fire all requests in parallel with `Promise.allSettled` (non-blocking, partial failures OK). Store results in a `Map<string, SessionRecord[]>` state. For each member row, show up to 3 small dots (w-2.5 h-2.5 rounded-full) in a row:

   - yellow dot: late
   - red dot: absent
   - gray dot: no data / not_marked
   If the API call is still loading, show 3 gray placeholder dots. This gives conductors instant pattern recognition.

If a note already exists, show the icon with a small blue dot indicator. The textarea auto-saves with the same debounce mechanism.
