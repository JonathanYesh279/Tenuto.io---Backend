**services/auditTrail.service.js:** Add a new exported function `logImpersonatedAction` to the service export object. This function logs mutating actions performed during impersonation:
```javascript
async function logImpersonatedAction(impersonationContext, req) {
  // impersonationContext = req.impersonation (set by enrichImpersonationContext middleware)
  // Only log mutating methods (POST, PUT, PATCH, DELETE)
  try {
    const collection = await getCollection(COLLECTIONS.PLATFORM_AUDIT_LOG);
    const entry = {
      action: AUDIT_ACTIONS.IMPERSONATION_ACTION,
      actorId: impersonationContext.superAdminId,
      actorType: 'super_admin',
      targetType: 'impersonated_action',
      targetId: impersonationContext.impersonatedUserId,
      details: {
        sessionId: impersonationContext.sessionId,
        tenantId: impersonationContext.tenantId,
        method: req.method,
        path: req.originalUrl,
        // Do NOT log request body (may contain sensitive data)
      },
      timestamp: new Date(),
      ip: req.ip || null,
    };
    await collection.insertOne(entry);
  } catch (err) {
    log.error({ err: err.message }, 'Failed to log impersonated action');
  }
}
```
Import `AUDIT_ACTIONS` from constants at the top. Add `logImpersonatedAction` to the export object.

**api/super-admin/super-admin.service.js:** Add two new functions and export them:

1. `startImpersonation(tenantId, superAdminId)`:
   - Validate tenantId format (ObjectId hex string)
   - Find tenant by `_id: ObjectId.createFromHexString(tenantId)`. If not found, throw `Tenant with id ${tenantId} not found`
   - If `!tenant.isActive`, throw `Cannot impersonate a deactivated tenant`
   - Find an active admin teacher: query `teacher` collection for `{ tenantId: tenant.tenantId || tenant._id.toString(), roles: 'מנהל', isActive: true }`. If none found, throw `No active admin found for this tenant`
   - Generate session ID: `new ObjectId().toString()`
   - Build token payload matching `generateAccessToken` output exactly (read auth.service.js to confirm field names), plus impersonation claims:
     ```
     { _id, tenantId, firstName, lastName, email, roles, version,
       isImpersonation: true,
       impersonatedBy: superAdminId,
       impersonationSessionId: sessionId }
     ```
   - Sign with `process.env.ACCESS_TOKEN_SECRET`, expiresIn `'1h'` (matching regular token TTL)
   - Log audit: `auditTrailService.logAction(AUDIT_ACTIONS.IMPERSONATION_STARTED, superAdminId, { targetId: tenantId, targetType: 'tenant', tenantName: tenant.name, impersonatedAdminId, impersonatedAdminEmail, sessionId })`
   - Return `{ accessToken, sessionId, tenant: { _id, name, slug }, impersonatedAdmin: { _id, name, email } }`

2. `stopImpersonation(sessionId, superAdminId)`:
   - Log audit: `auditTrailService.logAction(AUDIT_ACTIONS.IMPERSONATION_ENDED, superAdminId, { targetType: 'impersonation_session', targetId: sessionId, sessionId })`
   - Return `{ success: true, message: 'Impersonation session ended' }`

Both functions follow the existing service patterns (error handling, logging). Do NOT modify `authenticateToken` or `buildContext`.

**middleware/impersonation-audit.middleware.js (NEW FILE):**
Create this file with:
- Import `jwt` from `jsonwebtoken`
- Import `createLogger` from `../services/logger.service.js`
- Import `auditTrailService` from `../services/auditTrail.service.js`
- Export `enrichImpersonationContext` function:
  1. If `!req.teacher`, call `next()` (not an authenticated request)
  2. Extract token from `req.headers['authorization']?.split(' ')[1]`
  3. If no token, call `next()`
  4. `jwt.decode(token)` (NOT verify -- already verified by authenticateToken)
  5. If `decoded?.isImpersonation`:
     - Set `req.impersonation = { isImpersonation: true, superAdminId: decoded.impersonatedBy, sessionId: decoded.impersonationSessionId, impersonatedUserId: decoded._id, tenantId: decoded.tenantId }`
     - If request method is POST, PUT, PATCH, or DELETE: call `auditTrailService.logImpersonatedAction(req.impersonation, req)` (fire-and-forget, do NOT await -- audit must not slow down requests)
     - Log at debug level
  6. Call `next()`
  7. Wrap everything in try/catch -- on error, log warning and call `next()` (audit enrichment must never break requests)
