  <action>
Fix the date filtering in analytics/attendance.service.js to use ISO string comparison instead of BSON Date objects. Per Phase 76-02 decision: "activity_attendance.date stored as ISO string, not BSON Date — date filters must use string comparison."

There are two locations where `new Date()` is used for date filters:

1. Lines ~59-63 (getStudentAttendanceAnalytics):
```javascript
// BEFORE:
if (startDate) timeFilter.date.$gte = new Date(startDate);
if (endDate) timeFilter.date.$lte = new Date(endDate);

// AFTER:
if (startDate) timeFilter.date.$gte = new Date(startDate).toISOString();
if (endDate) timeFilter.date.$lte = new Date(endDate).toISOString();
```

2. Lines ~176-179 (getTeacherAttendanceAnalytics):
```javascript
// BEFORE:
if (startDate) filter.date.$gte = new Date(startDate);
if (endDate) filter.date.$lte = new Date(endDate);

// AFTER:
if (startDate) filter.date.$gte = new Date(startDate).toISOString();
if (endDate) filter.date.$lte = new Date(endDate).toISOString();
```

This ensures MongoDB's string comparison ($gte/$lte on strings) matches the ISO-string-stored date values correctly. ISO strings sort lexicographically in date order, so $gte/$lte work correctly on them.

Do NOT change any other logic or calculations in this file.
