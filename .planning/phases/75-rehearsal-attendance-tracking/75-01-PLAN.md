<objective>
Persist attendanceCount on rehearsal documents and wire frontend to prefer server values.

Purpose: The entire attendance API + UI already exists from v1.9 (Phases 59-60). The only gap is that `updateAttendance` writes `attendance: { present[], absent[], late[] }` but does NOT persist a denormalized `attendanceCount` field. The frontend `apiService.js` always client-computes attendanceCount from the attendance arrays, ignoring any server value. This means the Dashboard tooltip (already wired at Dashboard.tsx line 772) never shows data because it reads `attendanceCount` which is always recomputed as zeros when no attendance arrays come back from the list endpoint.

Output: Two small code changes — backend persists attendanceCount in transaction, frontend prefers server value with client fallback.
</objective>
