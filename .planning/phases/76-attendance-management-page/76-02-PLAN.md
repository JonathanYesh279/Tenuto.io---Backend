**Imports:**
- `attendanceAlertService`, `rehearsalService`, `orchestraService` from `../services/apiService`
- `GlassStatCard` from `../components/ui/GlassStatCard`
- `GlassSelect` from `../components/ui/GlassSelect`
- `SearchInput` from `../components/ui/SearchInput`
- `Table as HeroTable, TableHeader, TableColumn, TableBody, TableRow, TableCell, Pagination, Spinner, Button as HeroButton, Chip` from `@heroui/react`
- `Badge` from `../components/ui/badge`
- `AttendanceManager` from `../components/AttendanceManager`
- `Modal` from `../components/ui/Modal`
- `BarChart` from `../components/charts/HebrewCharts`
- `TableSkeleton` from `../components/feedback/Skeleton`
- `EmptyState` from `../components/feedback/EmptyState`
- `ErrorState` from `../components/feedback/ErrorState`
- Phosphor icons: `UsersIcon, ChartBarIcon, WarningIcon, CalendarIcon, MusicNoteIcon, TrendUpIcon, CaretRightIcon, CheckCircleIcon, ClockIcon`
- `useAuth` from `../services/authContext`

**State:**
- `dashboardData` — full response from `attendanceAlertService.getDashboard()` (perOrchestra, monthlyTrends, flaggedStudents, summary)
- `loading` / `error` — standard loading/error states
- `orchestraFilter` — selected orchestraId for filtering (empty = all)
- `dateRange` — `{ startDate, endDate }` defaulting to last 3 months (compute from `new Date()`)
- `searchQuery` — search string for orchestra names / student names
- `selectedOrchestra` — orchestraId for drill-down (shows rehearsals)
- `orchestraRehearsals` — rehearsals array for selected orchestra
- `rehearsalsLoading` — loading state for rehearsal fetch
- `selectedRehearsal` — rehearsal object for AttendanceManager modal
- `showAttendanceManager` — boolean toggle for modal
- `activeTab` — `'overview' | 'flagged' | 'trends'` for section switching

**Data Loading (useEffect):**
On mount and when dateRange changes, call `attendanceAlertService.getDashboard({ startDate: dateRange.startDate, endDate: dateRange.endDate })`. Store response in `dashboardData`.

**Page Layout (top to bottom):**

1. **Page Header:**
   ```
   <div className="flex items-center justify-between">
     <div>
       <h1 className="text-2xl font-bold text-foreground">ניהול נוכחות</h1>
       <p className="text-muted-foreground text-sm mt-1">מעקב נוכחות, התראות וסטטיסטיקות לכלל התזמורות</p>
     </div>
   </div>
   ```

2. **Stat Cards Row (GlassStatCard grid):**
   4 cards in `grid grid-cols-2 lg:grid-cols-4 gap-3`:
   - `שיעור נוכחות כללי` — value: `summary.overallAttendanceRate + '%'`, icon percentage/trend
   - `תזמורות פעילות` — value: `summary.totalOrchestras`
   - `תלמידים מסומנים` — value: `summary.totalFlagged`, use `valueClassName="text-red-600"` if > 0
   - `תלמידים במעקב` — value: `summary.totalStudentsTracked`
   All with `size="sm"`.

3. **Tab Navigation:**
   Three tabs styled as pill buttons (matching the toggle pattern from Teachers/TheoryLessons pages):
   - `סקירת תזמורות` (overview) — default
   - `תלמידים מסומנים` (flagged) — show badge with count if > 0
   - `מגמות חודשיות` (trends)

4. **Filter Bar (below tabs):**
   `<div className="flex items-center gap-3 flex-wrap">`
   - `GlassSelect` for orchestra filter: options from `dashboardData.perOrchestra` mapped to `{ value: orchestraId, label: orchestraName }` plus "כל התזמורות" default.
   - Date range: two `<input type="date">` for startDate and endDate, styled with border-border rounded px-3 py-1.5 text-sm.
   - `SearchInput` for text search (filters orchestra table by name or flagged students by name).

5. **Overview Tab — Per-Orchestra HeroUI Table:**
   Columns: `תזמורת` (orchestra name), `מנצח` (conductor), `חזרות` (total rehearsals), `חברים` (member count), `שיעור נוכחות` (attendance rate as percentage with color coding: green >= 80%, yellow 60-79%, red < 60%), `מסומנים` (flagged count, red chip if > 0), `פעולות` (expand/drill-down button).

   Data source: `dashboardData.perOrchestra`, filtered by orchestraFilter and searchQuery.

   Client-side pagination with HeroUI Pagination (10 per page).

   **Drill-down:** When user clicks the CaretRight action button on an orchestra row, set `selectedOrchestra` and fetch rehearsals via `rehearsalService.getOrchestraRehearsals(orchestraId)`. Show a collapsible section below the table (or replace table) listing recent rehearsals with: date, attendance count (from `attendanceCount.present` / `attendanceCount.total`), and a "סמן נוכחות" button that opens AttendanceManager.

6. **Flagged Tab — Flagged Students Section:**
   Render `dashboardData.flaggedStudents` in a HeroUI Table:
   Columns: `תלמיד` (studentName), `תזמורת` (orchestraName), `שיעור נוכחות` (attendanceRate%), `היעדרויות ברצף` (consecutiveAbsences), `סיבות` (flags rendered as Badge components).

   Badge rendering per flag:
   - `consecutive_absences` -> `variant="destructive"`, text: `${flag.value} היעדרויות ברצף`
   - `high_absence_rate` -> `variant="warning"`, text: `שיעור היעדרות ${flag.value}%`

   Filter by searchQuery on studentName. Filter by orchestraFilter on orchestraId.

7. **Trends Tab — Monthly Attendance Chart:**
   Render `dashboardData.monthlyTrends` using `BarChart` from HebrewCharts.tsx.
   Map data to: `{ label: month (format as Hebrew month name), value: attendanceRate, color: rate >= 80 ? green : rate >= 60 ? yellow : red }`.
   Show inside a card with title "מגמות נוכחות חודשיות".

8. **AttendanceManager Modal:**
   When `showAttendanceManager && selectedRehearsal`:
   ```tsx
   <AttendanceManager
     rehearsal={selectedRehearsal}
     orchestraId={selectedRehearsal.groupId}
     onSaved={() => { loadDashboard(); loadRehearsals(selectedOrchestra); }}
     onClose={() => { setShowAttendanceManager(false); setSelectedRehearsal(null); }}
   />
   ```

   **IMPORTANT:** Before opening AttendanceManager, ensure the rehearsal has `orchestra.members` populated. Fetch via `orchestraService.getOrchestra(orchestraId)` to get members, then merge: `{ ...rehearsal, orchestra: orchestraData }`. Follow the pattern from Rehearsals.tsx.

**Loading/Error/Empty states:**
- While loading: `<TableSkeleton rows={5} />` in the table area, stat cards with loading prop if supported.
- On error: `<ErrorState message="שגיאה בטעינת נתוני נוכחות" onRetry={loadDashboard} />`
- Empty perOrchestra: `<EmptyState title="אין נתוני נוכחות" description="לא נמצאו נתוני נוכחות לתקופה שנבחרה" />`

**Styling:** Use design system tokens (text-foreground, text-muted-foreground, bg-background, border-border). Follow the exact same spacing and layout as Orchestras.tsx. RTL is automatic via Tailwind plugin.
