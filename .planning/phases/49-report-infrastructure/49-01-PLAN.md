Create `api/reports/report.contract.js`:

1. Define COLUMN_TYPES constant: `{ STRING: 'string', NUMBER: 'number', DATE: 'date', PERCENTAGE: 'percentage', CURRENCY: 'currency' }`.

2. `validateGeneratorOutput(output)` — validates a generator's raw return value has the required shape:
   - `columns` must be an array of objects with at least `{ key, label, type }` where type is a valid COLUMN_TYPES value. `sortable` defaults to true, `format` is optional.
   - `rows` must be an array (can be empty for empty state).
   - `summary` must be an object with `items` array, each item having `{ label, value, type }`. `trend` (with `delta`, `direction`) is optional.
   - `columnGroups` is optional; if present, must be an array of `{ label, columns: [keys] }`.
   - `comparisonRows` is optional; if present, must be an array matching rows shape.
   - Returns `{ valid: true }` or `{ valid: false, errors: ['...'] }`.

3. `shapeResponse(reportMeta, generatorOutput, paginationParams)` — takes generator metadata (id, title, supported exports), the validated generator output, and pagination info `{ page, limit, totalCount }` to produce the final API response shape:
   ```
   {
     metadata: {
       reportId, title, generatedAt (ISO string),
       pagination: { page, limit, totalCount, totalPages: Math.ceil(totalCount/limit) },
       filters: { ...appliedFilters },
       exports: ['excel', 'pdf']  // from generator meta
     },
     columns: [...],
     columnGroups: [...] || undefined,
     rows: [...],
     summary: { items: [...] },
     comparisonRows: [...] || undefined
   }
   ```

Create `api/reports/report.scope.js`:

1. `buildReportScope(context)` — reads `req.context` fields to produce a scope object for generators:
   - If `context.isAdmin` is true: `{ type: 'all', tenantId: context.tenantId }`
   - If `context.isCoordinator` is true: `{ type: 'department', tenantId: context.tenantId, departmentIds: context.coordinatorDepartments }`
   - Otherwise (teacher): `{ type: 'own', tenantId: context.tenantId, teacherId: context.userId }`
   - Always includes `tenantId` from context.

Note: Teachers should never reach reports (blocked by requirePermission), but the scope builder handles them defensively. The orchestrator (plan 02) will use this scope and pass it to generators so generators are role-unaware.

Create `api/reports/generators/_example.generator.js`:

This is a reference/stub generator used for pipeline testing. It follows the exact generator plugin convention from the decisions:

```js
export default {
  id: '_example',
  name: 'Example Report',
  description: 'Reference stub for pipeline testing',
  category: 'institutional',
  icon: 'FileText',
  roles: ['מנהל', 'סגן מנהל', 'מזכירות'],  // admin-tier only
  params: {
    // Report-specific params declaration (beyond shared params)
    // Each param: { type, required, default, allowed }
    groupBy: { type: 'string', required: false, default: 'none', allowed: ['none', 'department'] }
  },
  columns: [
    { key: 'name', label: 'שם', type: 'string', sortable: true },
    { key: 'count', label: 'כמות', type: 'number', sortable: true },
    { key: 'percentage', label: 'אחוז', type: 'percentage', sortable: true },
  ],
  exports: ['excel', 'pdf'],
  async generate(params, scope, { services }) {
    // Stub: returns static data for pipeline verification
    const rows = [
      { name: 'Example A', count: 10, percentage: 50 },
      { name: 'Example B', count: 8, percentage: 40 },
      { name: 'Example C', count: 2, percentage: 10 },
    ];
    return {
      columns: this.columns,
      rows,
      summary: {
        items: [
          { label: 'סה"כ', value: 20, type: 'number' },
          { label: 'ממוצע', value: 6.67, type: 'number' },
        ]
      }
    };
  }
};
```

Create `api/reports/report.registry.js`:

1. Import `fs` and `path`, `fileURLToPath` for ESM __dirname.
2. Import `createLogger` from `../../services/logger.service.js`.

3. Module-level state: `let generators = new Map()` and `let loaded = false`.

4. `loadGenerators()` — called once at startup:
   - Reads all `*.generator.js` files from the `generators/` subdirectory using `fs.readdirSync`.
   - For each file, dynamically imports it (`await import(...)`) and reads the default export.
   - Validates the generator has required fields: `id`, `name`, `category`, `roles`, `generate` (function).
   - Stores in the Map keyed by `generator.id`.
   - Logs count of loaded generators.
   - Skips files starting with `_` (like `_example`) in PRODUCTION (process.env.NODE_ENV === 'production'). In dev/test, loads all including `_example`.

5. `getRegistry(userRoles)` — returns array of generator metadata objects (without the generate function) filtered by role:
   - For each generator in the Map, check if any of the user's roles appears in `generator.roles`.
   - Return array of `{ id, name, description, category, icon, params, columns, exports }` for matching generators.
   - Sort by category then name.

6. `getGenerator(reportId)` — returns the full generator object by id, or null if not found.

7. Export `{ loadGenerators, getRegistry, getGenerator }`.
