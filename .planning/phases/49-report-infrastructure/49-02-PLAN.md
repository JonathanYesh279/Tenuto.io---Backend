  <name>Task 1: Orchestrator with param validation, pagination, sorting, and year-over-year</name>
  <files>api/reports/report.orchestrator.js</files>
  <action>
Create `api/reports/report.orchestrator.js`:

Import from plan 01 artifacts: `getRegistry`, `getGenerator` from `./report.registry.js`; `validateGeneratorOutput`, `shapeResponse` from `./report.contract.js`; `buildReportScope` from `./report.scope.js`.
Import `createLogger` from `../../services/logger.service.js`.

Export `reportOrchestrator` object with these methods:

1. `getAvailableReports(context)`:
   - Calls `getRegistry(context.userRoles)` to get role-filtered report list.
   - Returns the array directly.

2. `generateReport(reportId, queryParams, context)`:
   - **Resolve generator:** `getGenerator(reportId)`. If null, throw error with code `REPORT_NOT_FOUND`.
   - **Role check:** Verify at least one of `context.userRoles` is in `generator.roles`. If not, throw error with code `REPORT_ACCESS_DENIED`.
   - **Parse shared params** from queryParams:
     - `page`: integer, default 1, min 1
     - `limit`: integer, default 50, min 1, max 500
     - `sortBy`: string, default null (generator decides)
     - `sortOrder`: 'asc' or 'desc', default 'asc'
     - `schoolYearId`: string, default `context.schoolYearId` (from req.context, which comes from addSchoolYearToRequest middleware)
     - `comparisonYearId`: string, optional (triggers comparison data)
     - `department`: string, optional (additional filter)
   - **Validate report-specific params:** For each param declared in `generator.params`, check the matching queryParam value against `{ type, required, allowed }`. Collect validation errors. If any, throw error with code `INVALID_PARAMS` and errors array.
   - **Build scope:** `buildReportScope(context)`.
   - **Build params object** combining shared + report-specific params.
   - **Build services object** to inject into generator: `{ getCollection }` from `../../services/mongoDB.service.js`. Generators use existing service functions, but for flexibility also pass getCollection. Future phases may expand this.
   - **Call generator:** `const rawOutput = await generator.generate(params, scope, { services })`. Wrap in try/catch — generator errors get code `GENERATOR_ERROR`.
   - **Validate output:** `validateGeneratorOutput(rawOutput)`. If invalid, log the errors and throw with code `INVALID_GENERATOR_OUTPUT`.
   - **Apply sorting:** If `sortBy` is provided and matches a column key with `sortable: true`, sort the rows array in-memory. Use locale-aware comparison for strings (`localeCompare` with 'he' locale), numeric comparison for numbers/percentage/currency. Apply `sortOrder`.
   - **Apply pagination:** Slice the sorted rows: `totalCount = rows.length`, then `rows.slice((page-1)*limit, page*limit)`.
   - **Shape response:** Call `shapeResponse()` with generator metadata (`{ id: generator.id, title: generator.name, exports: generator.exports }`), the paginated output, and `{ page, limit, totalCount }`. Also pass the applied filters (schoolYearId, department, any report-specific params).
   - **Return** the shaped response.
   - **Timing:** Log execution time using `Date.now()` before and after generator call.

Error objects thrown should have shape `{ message, code, status, errors? }` where status is the HTTP status code (404, 403, 400, 500).
  </action>

Create `api/reports/report.controller.js`:

Follow the same pattern as `api/hours-summary/hours-summary.controller.js` (import service, export controller object, async functions with try/catch/next).

1. `getRegistry(req, res, next)`:
   - Calls `reportOrchestrator.getAvailableReports(req.context)`.
   - Returns `res.json({ reports: [...] })`.

2. `getReport(req, res, next)`:
   - Reads `req.params.reportId` and `req.query` (all query params).
   - Calls `reportOrchestrator.generateReport(reportId, req.query, req.context)`.
   - Returns `res.json(result)` (already shaped by orchestrator).
   - In catch: if error has `.status`, use it; otherwise 500. Return `{ error: err.message, code: err.code, errors: err.errors }`.

Create `api/reports/report.route.js`:

Follow the pattern from `api/hours-summary/hours-summary.route.js`:

```js
import express from 'express';
import { reportController } from './report.controller.js';
import { requirePermission } from '../../middleware/auth.middleware.js';

const router = express.Router();

// Get available reports catalog filtered by user role
router.get('/registry', requirePermission('reports', 'view'), reportController.getRegistry);

// Generate a specific report with params
router.get('/:reportId', requirePermission('reports', 'view'), reportController.getReport);

export default router;
```

Uses `requirePermission('reports', 'view')` — this is the existing RBAC permission domain. Admin-tier gets `all` scope, coordinators get `department` scope per the permission matrix in config/permissions.js. Teachers have no `reports.view` permission by default, so they are blocked at the middleware level.

Modify `server.js`:

1. Add import at the top with other route imports: `import reportRoutes from './api/reports/report.route.js';`
2. Add import for registry init: `import { loadGenerators } from './api/reports/report.registry.js';`
3. Add route mount following the standard middleware chain pattern (same as hours-summary):
   ```js
   app.use(
     '/api/reports',
     authenticateToken,
     enrichImpersonationContext,
     buildContext,
     enforceTenant,
     stripTenantId,
     addSchoolYearToRequest,
     reportRoutes
   );
   ```
   Place this AFTER the hours-summary mount and BEFORE the super-admin mount.
4. Call `await loadGenerators()` during server startup — add it inside the existing startup sequence (after DB connection, before `app.listen` or similar). Find the pattern used for other startup tasks and follow it. If the server uses a top-level async function or `.then()` chain, add it there.

Note: `addSchoolYearToRequest` is included in the chain so that `req.schoolYear` and `req.schoolYearId` are available, and `buildContext` populates `req.context.schoolYearId` for the orchestrator to use as default.
