**RISK-ASSESSMENT.md** -- Formal risk assessment document. Structure:

1. **Document header:** Title ("Information Security Risk Assessment"), version 1.0, date, classification "INTERNAL -- COMPLIANCE DOCUMENT".

2. **Methodology:** Describe the Likelihood x Impact matrix approach (per user decision: 3x3 grid). Define:
   - Likelihood levels: Low (unlikely, requires multiple failures), Medium (possible, known attack vector exists), High (probable, actively exploitable or no controls in place)
   - Impact levels: Low (minimal data exposure, easily contained), Medium (significant data exposure affecting multiple records), High (mass data breach, minors' data exposed, regulatory penalties)
   - Risk Level calculation: L x I matrix yielding LOW / MEDIUM / HIGH / CRITICAL risk levels

3. **Risk matrix visualization:** A visual 3x3 grid (using markdown table or ASCII art) showing the mapping:
   ```
                    Impact
                    Low      Medium    High
   Likelihood High  MEDIUM   HIGH      CRITICAL
              Med   LOW      MEDIUM    HIGH
              Low   LOW      LOW       HIGH
   ```
   Note: Low likelihood + High impact = HIGH (because minors' data breach consequences are severe)

4. **Risk register:** For ALL 12 risks from research section 4.1, create a detailed entry. For each risk:
   - Risk ID (R-01 through R-12)
   - Threat description
   - Likelihood: Low / Medium / High with justification
   - Impact: Low / Medium / High with justification
   - Risk Level (from matrix)
   - Existing controls (what the platform already has)
   - Recommended mitigations (what should be implemented)
   - Target phase for remediation: "v1.6 Technical Hardening" for all technical recommendations (per user decision and specific ideas)
   - Residual risk after mitigation

   Use the exact risks from research:
   - R-01: Cross-tenant data leak
   - R-02: Credential exposure in database dump
   - R-03: Minors' data breach via API
   - R-04: JWT secret compromise
   - R-05: Default password exploitation
   - R-06: Import log PII retention
   - R-07: Deletion snapshot PII accumulation
   - R-08: Insufficient logging for compliance
   - R-09: SendGrid/email data in transit to US
   - R-10: S3 bucket misconfiguration
   - R-11: No data retention enforcement
   - R-12: Student name denormalization in teacher docs

5. **Risk summary table:** Consolidated table: Risk ID | Threat | Likelihood | Impact | Risk Level | Status (all "Open -- remediation planned for v1.6")
