  <files>api/super-admin/super-admin.service.js, api/super-admin/super-admin.validation.js</files>
  <action>
Add four new functions to superAdminService in super-admin.service.js:

1. `getTenantAdmins(tenantId)`:
   - Query teacher collection for `{ tenantId, roles: { $in: ADMIN_TIER_ROLES }, isActive: true }`
   - Project: `_id, personalInfo.firstName, personalInfo.lastName, personalInfo.email, credentials.email, credentials.requiresPasswordChange, credentials.lastLogin, credentials.invitationMode, credentials.passwordSetAt, roles, createdAt, updatedAt`
   - Exclude password, refreshToken, tokenVersion from projection
   - Sort by `personalInfo.firstName` ascending
   - Return array of admin teacher docs

2. `getAllTenantAdmins()`:
   - Query ALL teachers with `{ roles: { $in: ADMIN_TIER_ROLES }, isActive: true }` across all tenants
   - Same projection as above, but also include `tenantId`
   - Also do a tenant lookup to add `tenantName` and `tenantSlug` to each result
   - Approach: query teachers, then batch-lookup tenants by unique tenantIds, merge tenant name/slug into each result
   - Sort by tenantName then firstName

3. `updateTenantAdmin(tenantId, adminId, data)`:
   - Validate data with `updateTenantAdminSchema` (see below)
   - Find teacher by `{ _id: ObjectId(adminId), tenantId, roles: { $in: ADMIN_TIER_ROLES } }`
   - If not found, throw "Admin teacher not found" error
   - Build $set from validated fields: `personalInfo.firstName`, `personalInfo.lastName`, `personalInfo.email` (also update `credentials.email` to keep in sync), `updatedAt`
   - If email changed, check uniqueness within tenant: `{ tenantId, 'credentials.email': newEmail, _id: { $ne: ObjectId(adminId) } }`
   - Return updated doc (without password/token fields)

4. `resetTenantAdminPassword(tenantId, adminId)`:
   - Find teacher by `{ _id: ObjectId(adminId), tenantId, roles: { $in: ADMIN_TIER_ROLES } }`
   - If not found, throw "Admin teacher not found" error
   - Hash DEFAULT_PASSWORD with bcrypt (SALT_ROUNDS = 10)
   - Update: `credentials.password = hashed, credentials.requiresPasswordChange = true, credentials.passwordSetAt = new Date(), updatedAt = new Date()`
   - Log the action
   - Return `{ success: true, message: 'Password reset to default' }`

Export all four from the superAdminService object.

In super-admin.validation.js, add:
```js
export const updateTenantAdminSchema = Joi.object({
  firstName: Joi.string().trim(),
  lastName: Joi.string().trim(),
  email: Joi.string().email(),
}).min(1);
```
