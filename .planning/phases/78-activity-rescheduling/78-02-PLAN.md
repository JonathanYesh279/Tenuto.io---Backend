<objective>
Wire the frontend ActivityDetailModal to show Day selector for all activity types, implement client-side conflict preview before save, and pass day changes to the correct backend APIs with proper date calculation.

Purpose: Currently the Day selector only appears for lessons (timeBlock with lessonId). This plan makes it universal, adds real-time conflict preview using loaded schedule data, and wires each source type's save handler to pass day changes to the appropriate API — including calculating a correct target date when changing rehearsal days.

Design decision — Rehearsal pattern-vs-instance: This implementation updates a SINGLE rehearsal instance (the one clicked in the modal). Room schedule deduplicates rehearsals by pattern ($group by {groupId, dayOfWeek, startTime, endTime, location}), so changing one instance's day while siblings remain will split the display. This is the correct behavior for "reschedule this specific rehearsal" — the user is moving one occurrence, not changing the recurring pattern. If bulk pattern updates are needed later, that's a separate feature (bulk edit from Orchestra details page). The modal title/UX should make clear this is a single-instance operation.

Output: ActivityDetailModal with universal Day selector, real-time conflict preview, and full day/room/time rescheduling for all activity types.
</objective>

3. **Add conflict check effect:** When `editDay`, `editRoom`, `editStartTime`, or `editEndTime` changes, check for conflicts:
   ```
   useEffect(() => {
     if (!activity || !open) return

     const checkConflicts = async () => {
       setCheckingConflicts(true)
       try {
         // Get schedule for the target day
         let targetSchedule = scheduleData
         if (editDay !== day && getScheduleForDay) {
           targetSchedule = await getScheduleForDay(editDay)
         }

         if (!targetSchedule) {
           setPreviewConflicts([])
           return
         }

         // Find the target room's activities
         const roomData = targetSchedule.rooms.find(r => r.room === editRoom)
         if (!roomData) {
           setPreviewConflicts([])
           return
         }

         // Filter out the current activity and check for time overlaps
         const conflicts = roomData.activities
           .filter(a => a.id !== activity.id)  // exclude self
           .filter(a => doTimesOverlap(editStartTime, editEndTime, a.startTime, a.endTime))
           .map(a => ({
             room: editRoom,
             startTime: a.startTime,
             endTime: a.endTime,
             title: a.teacherName || a.title || 'פעילות',
             source: a.source || 'unknown',
           }))

         setPreviewConflicts(conflicts)
       } catch {
         setPreviewConflicts([])  // fail silently — server-side check is the safety net
       } finally {
         setCheckingConflicts(false)
       }
     }

     // Debounce to avoid excessive API calls when user is still selecting
     const timer = setTimeout(checkConflicts, 300)
     return () => clearTimeout(timer)
   }, [editDay, editRoom, editStartTime, editEndTime, activity, open, day, scheduleData, getScheduleForDay])
   ```

   Import `doTimesOverlap` from `./utils`.
