   ```javascript
   })).optional().default([])
   ```
   Add a standalone room validation schema for add/update:
   ```javascript
   export const roomSchema = Joi.object({
     name: Joi.string().trim().min(1).max(100).required(),
   });
   export function validateRoom(data) {
     return roomSchema.validate(data, { abortEarly: false });
   }
   ```
   IMPORTANT: The `rooms` field must be `.optional()` in both schemas so existing settings updates (which don't send rooms) continue to work. Add `.default([])` only in the create schema, not the update schema.

   - `getRooms(tenantId)` -- findOne with projection `{ 'settings.rooms': 1 }`, return `tenant.settings.rooms || []`
   - `addRoom(tenantId, roomData)` -- validate with `validateRoom`, normalize name via `name.trim().replace(/\s+/g, ' ')`, check duplicate name (case-sensitive, normalized), use `$push` to add `{ _id: new ObjectId(), name, isActive: true, createdAt: new Date() }` to `settings.rooms`, also `$set: { updatedAt: new Date() }`. Return the new room object.
   - `updateRoom(tenantId, roomId, roomData)` -- validate with `validateRoom`, normalize name, check duplicate name excluding current roomId, use `findOneAndUpdate` with `{ 'settings.rooms._id': ObjectId }` filter and `$set: { 'settings.rooms.$.name': normalizedName, updatedAt: new Date() }`. Return updated tenant.settings.rooms.
   - `deactivateRoom(tenantId, roomId)` -- use `findOneAndUpdate` with positional operator to set `settings.rooms.$.isActive` to false. Return updated rooms array.

   Follow existing error patterns: throw `new Error('Room with name "X" already exists')` for duplicates, throw `new Error('Room not found')` when roomId doesn't match.
