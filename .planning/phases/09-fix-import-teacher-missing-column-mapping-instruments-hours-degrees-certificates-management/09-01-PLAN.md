**A. Replace hardcoded instrument column threshold with dynamic section detection (Bug 1 + Bug 2)**

In `detectInstrumentColumns()` (line ~530), replace the `colIndex < 24` hardcoded threshold:

1. First pass: scan ALL headers to find the EARLIEST column index where a header matches `ABBREVIATION_TO_INSTRUMENT` or `DEPARTMENT_TO_INSTRUMENTS`. Store as `instrumentSectionStart`.
2. Second pass: collect instrument columns at or after `instrumentSectionStart`.
3. To avoid the "פסנתר" at accomp-hours-column problem, maintain a set of known hours-column header names that happen to share names with instruments. Use the existing `HOURS_FIELDS_WITH_ROLE_COLLISION` pattern -- create a `KNOWN_NON_INSTRUMENT_HEADERS` set containing the TEACHER_COLUMN_MAP keys that map to hours fields: `new Set(['הוראה', 'ליווי פסנתר', 'הרכב ביצוע', 'להרכב ביצוע', 'ריכוז הרכב', 'תאוריה', 'תיאוריה', 'ניהול', 'ריכוז', 'ביטול זמן', 'סה"כ ש"ש', "סה''כ ש''ש", 'שבועיות', 'זמן', 'שעות'])`.
4. In the second pass, skip any header that is in `KNOWN_NON_INSTRUMENT_HEADERS` AND whose column index is BEFORE `instrumentSectionStart` (i.e., it's in the hours section, not the instrument matrix). If a header like "ליווי פסנתר" appears BOTH before and after instrumentSectionStart, the one before is hours, the one after is the role column.
5. If `instrumentSectionStart === Infinity` (no instruments found at all), return empty array.

Also fix Bug 2 (parent-row instrument abbreviations): In the header backfill logic (lines ~316-336), the current code only fills EMPTY columns from parent rows. Add a second pass specifically for instrument detection: after the main header backfill, scan parent rows (headerRowIndex-1 down to 0) for cells that match `ABBREVIATION_TO_INSTRUMENT` or `DEPARTMENT_TO_INSTRUMENTS`. If such a cell is found at column `c` and `headers[c]` does NOT already map to a known column in `TEACHER_COLUMN_MAP`, replace `headers[c]` with the parent-row instrument abbreviation. This ensures instrument abbreviations in parent rows are used even when the header row has a non-matching value.

**B. Guard sub-header refinement to not overwrite valid parent headers (Bug 4)**

In the sub-header refinement loop (line ~439), where `columnMap[subText]` triggers header replacement:

Change:
```javascript
if (columnMap[subText]) {
  headers[c] = subText;
  continue;
}
```

To:
```javascript
if (columnMap[subText]) {
  // Only replace if current header does NOT already map to a valid field,
  // OR the sub-header maps to a MORE SPECIFIC field than the current header.
  // This prevents overwriting correct headers like "סיווג" or "תואר"
  // with generic sub-header fragments.
  if (!columnMap[headers[c]]) {
    headers[c] = subText;
  }
  continue;
}
```

This ensures that if `headers[c]` already maps to a known column (e.g., "סיווג" -> "classification"), it is NOT replaced by a sub-header value. Only unmapped headers get replaced by sub-header values.

**C. Disambiguate "כן-לא" using parent-row context (Bug 5)**

In the sub-header refinement loop, add special handling for the generic "כן-לא" and "כן / לא" patterns BEFORE the `columnMap[subText]` check:

```javascript
if (subText === 'כן-לא' || subText === 'כן / לא') {
  // Check parent rows for context to determine which boolean field this is
  for (let r = headerRowIndex - 1; r >= Math.max(0, headerRowIndex - 3); r--) {
    const parentRow = allTextRows[r];
    if (parentRow?.[c]) {
      const parentText = parentRow[c].replace(/[\u200F\u200E\uFEFF\u200B]/g, '').trim();
      if (parentText.includes('תעודת הוראה') || parentText.includes('תעודה')) {
        headers[c] = 'תעודת הוראה';
        break;
      } else if (parentText.includes('ארגון') || parentText.includes('חבר')) {
        headers[c] = 'חבר ארגון';
        break;
      }
    }
  }
  continue; // Skip the generic columnMap lookup for "כן-לא"
}
```

This prevents the generic "כן-לא" from blindly mapping to `teachingCertificate` and instead uses the parent row to determine the correct field.

Note: Keep the existing `'כן-לא': 'teachingCertificate'` entry in TEACHER_COLUMN_MAP as a fallback for files without multi-row headers. The sub-header disambiguation above only runs during sub-header refinement.
