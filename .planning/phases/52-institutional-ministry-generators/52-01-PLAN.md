Create `year-over-year-comparison.generator.js` following the established plugin convention.

Generator metadata:
- id: 'year-over-year-comparison'
- name: 'השוואה שנתית'
- description: 'השוואת מדדים מרכזיים בין שנות לימודים'
- category: 'institutional'
- icon: 'TrendUp'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params: { comparisonYearId: { type: 'string', required: true } } (the primary schoolYearId comes from the standard params)
- exports: ['excel', 'pdf']

Columns:
- metric (string, sortable) — Hebrew metric name
- currentValue (number, sortable) — value for primary school year
- comparisonValue (number, sortable) — value for comparison school year
- change (number, sortable) — absolute difference (current - comparison)
- changePercent (number, sortable) — percentage change

generate() logic:
1. Get schoolYearId from params (primary year) and comparisonYearId (comparison year).
2. For both years, query these collections with tenantId filter:
   - `student` collection: count docs where `isActive: true` (and the school year filter — students don't have schoolYearId directly, so count ALL active students for current, and for historical comparison just count all active). Actually, students persist across years. Instead, count total students and active students from `student` collection (these are current state). For the comparison year, use `hours_summary` to get teacher/hour counts per year.

   Better approach: collect metrics from collections that ARE year-scoped:
   - `hours_summary`: filter by schoolYearId + tenantId. Count distinct teacherIds, sum totalWeeklyHours.
   - `orchestra`: filter by schoolYearId + tenantId. Count orchestras.
   - `student`: count by tenantId + isActive (not year-scoped, so show current count for both — or skip student count from year-scoped and use hours_summary student counts if available).

   Simplest reliable approach:
   - **Teachers with hours**: count distinct teacherIds from hours_summary per year
   - **Total weekly hours**: sum totalWeeklyHours from hours_summary per year
   - **Orchestras**: count from orchestra collection filtered by schoolYearId per year
   - **Students (active)**: count from student collection (tenantId only, not year-scoped — same for both years, so label accordingly OR count students who have teacherAssignments, which indicates enrollment)
   - **Hours summaries**: count docs in hours_summary per year

   For each metric, build a row with currentValue, comparisonValue, change, changePercent.

3. Fetch school year names from school_year collection to include in summary.

4. Summary items: primary year name, comparison year name, number of metrics improved, number declined.

Scope filtering: Only 'all' scope makes sense for institutional reports. For 'department' scope, filter hours_summary by teachers in those departments (same pattern as teacher-hours-summary). For 'own' scope, return empty (institutional report not meaningful for individual teacher).

Handle edge cases: if comparisonYearId not provided, return error-like empty result with message. If a year has no data, show 0s.
  </action>

<task type="auto">
  <name>Task 2: Import History generator (INST-04)</name>
  <files>api/reports/generators/import-history.generator.js</files>
  <action>
Create `import-history.generator.js` following the established plugin convention.

Generator metadata:
- id: 'import-history'
- name: 'היסטוריית ייבוא'
- description: 'סיכום פעולות ייבוא נתונים עם סטטוסים ותוצאות'
- category: 'institutional'
- icon: 'Upload'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params: { importType: { type: 'string', required: false, allowed: ['all', 'teachers', 'students', 'conservatory', 'ensembles'], default: 'all' } }
- exports: ['excel', 'pdf']

Columns:
- importType (string, sortable) — Hebrew type name (מורים/תלמידים/קונסרבטוריון/הרכבים)
- status (string, sortable) — Hebrew status (ממתין/בעיבוד/הושלם/חלקי/נכשל)
- createdAt (date, sortable) — import date
- uploadedBy (string, sortable) — who uploaded
- matchedCount (number, sortable) — successfully matched records
- notFoundCount (number, sortable) — unmatched records
- errorCount (number, sortable) — error count
- totalRecords (number, sortable) — total records in import

generate() logic:
1. Query `import_log` collection with tenantId filter. If importType param !== 'all', add importType filter.
2. Sort by createdAt descending.
3. Map each doc to a row:
   - importType: translate to Hebrew using a lookup map { teachers: 'מורים', students: 'תלמידים', conservatory: 'קונסרבטוריון', ensembles: 'הרכבים' }
   - status: translate to Hebrew { pending: 'ממתין', processing: 'בעיבוד', completed: 'הושלם', partial: 'חלקי', failed: 'נכשל' }
   - createdAt: doc.createdAt
   - uploadedBy: look up teacher name if uploadedBy is a teacherId, or use raw value. For simplicity, just output the stored value (it may already be a name or ID).
   - matchedCount: doc.preview?.matched?.length || doc.results?.matched || 0
   - notFoundCount: doc.preview?.notFound?.length || doc.results?.notFound || 0
   - errorCount: doc.preview?.errors?.length || doc.results?.errors || 0
   - totalRecords: matchedCount + notFoundCount + errorCount (or from doc.preview?.totalRows)

4. Summary items:
   - Total imports count
   - Completed count (status === 'completed')
   - Failed count (status === 'failed')
   - Last import date (most recent createdAt)

Scope: Institutional report — 'own' scope returns empty. 'all' and 'department' both show full import history (imports aren't department-scoped).
  </action>
