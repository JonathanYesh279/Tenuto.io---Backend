Create `ministry-readiness-audit.generator.js` following the established plugin convention.

Generator metadata:
- id: 'ministry-readiness-audit'
- name: 'ביקורת מוכנות למשרד'
- description: 'אחוז השלמה, שדות חסרים ושגיאות הצלבה לדוח המשרד'
- category: 'institutional'
- icon: 'ClipboardCheck'
- roles: ['מנהל', 'סגן מנהל']
- params: {} (uses standard schoolYearId from params)
- exports: ['excel', 'pdf']

Columns:
- entityType (string, sortable) — entity type in Hebrew (מורה/תלמיד/הרכב/קונסרבטוריון)
- entityName (string, sortable) — name of entity with missing data
- missingField (string, sortable) — Hebrew field name that is missing
- severity (string, sortable) — חובה (required) or אזהרה (warning)

generate() logic:
1. Import `exportService` from `../../export/export.service.js` at the top of the file (static import).
2. Call `exportService.getCompletionStatus(params.schoolYearId, { context: { tenantId: scope.tenantId } })` to get completionPercentage, missing array, preExportWarnings, preExportErrors, and counts.
3. Call `exportService.crossValidate(params.schoolYearId, { context: { tenantId: scope.tenantId } })` to get cross-validation warnings and errors.
4. Build rows from the `missing` array returned by getCompletionStatus:
   - Each missing entry has { type, name, field }. Map type to Hebrew entityType, name stays as-is, field is already Hebrew, severity = 'חובה'.
5. Add rows from crossValidate warnings/errors:
   - Each has { type, message }. Map to: entityType based on type (hours_mismatch -> 'שעות', missing_conductor -> 'הרכב', etc.), entityName = extract from message or use type, missingField = message, severity = errors get 'חובה', warnings get 'אזהרה'.
6. Add rows from preExportErrors/preExportWarnings similarly.

Summary items:
- Completion percentage (from getCompletionStatus)
- Total missing fields count
- Cross-validation warnings count
- Cross-validation errors count
- Entity counts (teachers, students, orchestras from counts object)

Scope: Institutional report. 'own' scope returns empty. 'all' and 'department' both run full audit (Ministry readiness is institution-wide). Note: exportService functions expect context with tenantId, which we construct from scope.tenantId.

Edge case: If schoolYearId not provided, try to proceed without it (getCompletionStatus handles null schoolYearId).
  </action>
  <verify>
Import the generator and verify: (1) default export has all required fields, (2) imports exportService, (3) generate calls getCompletionStatus and crossValidate, (4) columns match the 4-column spec.
  </verify>

<task type="auto">
  <name>Task 2: Data Quality Report generator (INST-03)</name>
  <files>api/reports/generators/data-quality.generator.js</files>
  <action>
Create `data-quality.generator.js` following the established plugin convention.

Generator metadata:
- id: 'data-quality'
- name: 'איכות נתונים'
- description: 'זיהוי חריגות: תלמידים ללא מורים, מורים ללא תלמידים, רשומות חסרות'
- category: 'institutional'
- icon: 'ShieldCheck'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params: { anomalyType: { type: 'string', required: false, default: 'all', allowed: ['all', 'unassigned-students', 'idle-teachers', 'incomplete-records', 'empty-orchestras'] } }
- exports: ['excel', 'pdf']

Columns:
- anomalyType (string, sortable) — Hebrew anomaly category
- entityType (string, sortable) — מורה/תלמיד/הרכב
- entityName (string, sortable) — name
- detail (string, sortable) — specific issue description in Hebrew
- severity (string, sortable) — גבוהה/בינונית/נמוכה

generate() logic:
1. Query collections: student, teacher, orchestra (all with tenantId filter, isActive: true).
2. Build anomaly rows for each category:

**Unassigned students** (anomalyType: 'תלמידים ללא שיבוץ'):
- Students where teacherAssignments is empty or missing.
- severity: 'גבוהה', detail: 'תלמיד/ה ללא שיבוץ למורה'

**Idle teachers** (anomalyType: 'מורים ללא תלמידים'):
- Collect all teacherIds from all active students' teacherAssignments. Teachers NOT in this set are idle.
- severity: 'בינונית', detail: 'מורה ללא תלמידים משובצים'

**Incomplete student records** (anomalyType: 'רשומות חלקיות'):
- Students missing key fields: academicInfo.class, academicInfo.studyYears, instrumentProgress (empty array or missing), personalInfo.phone or personalInfo.parentPhone (contact info).
- One row per missing field per student.
- severity: 'נמוכה' for contact info, 'בינונית' for academic fields.
- detail: 'חסר: {field name in Hebrew}'

**Empty orchestras** (anomalyType: 'הרכבים ריקים'):
- Orchestras where members array is empty or missing, or no students have this orchestraId in their assignments.
- Actually simpler: query orchestra collection, check if memberIds (or equivalent) is empty. If orchestra collection stores members differently, just check orchestra.students or orchestra.memberIds.
- Alternative: For each orchestra, count students that belong to it by checking student docs with matching orchestraId in their orchestra-related fields. But that's expensive. Instead, just check orchestra.members or orchestra.studentIds array length.
- If orchestra has no conductorId, add row too: 'הרכב ללא מנצח/מדריך'.
- severity: 'בינונית'

3. If anomalyType param !== 'all', filter rows to only that category.

4. Summary items:
   - Total anomalies found
   - High severity count
   - Medium severity count
   - Low severity count

Scope: 'all' shows everything. 'department' filters students/teachers by department (use getInstrumentDepartment pattern from other generators). 'own' scope returns empty.

Import getInstrumentDepartment from config/constants.js for department scope filtering.
