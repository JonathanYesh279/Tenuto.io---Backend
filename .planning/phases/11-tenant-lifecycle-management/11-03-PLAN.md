  **Create `services/tenantPurge.service.js`:**
  1. Import `getDB` from `./mongoDB.service.js`, `ObjectId` from `mongodb`, `createLogger`, `TENANT_SCOPED_COLLECTIONS` and `COLLECTIONS` from `../config/constants.js`
  2. Export `tenantPurgeService` with methods:

  `async previewDeletion(tenantId)`:
  - Get db via `getDB()`
  - For each collection in `TENANT_SCOPED_COLLECTIONS`, run `db.collection(name).countDocuments({ tenantId })`
  - Run all countDocuments in parallel using `Promise.all`
  - Return `{ counts: { teacher: N, student: N, ... }, total: sum }`

  `async createTenantSnapshot(tenantId)`:
  - Get db via `getDB()`
  - Create a unique snapshotId: `new ObjectId()`
  - For each collection in `TENANT_SCOPED_COLLECTIONS`:
    - Find all docs: `db.collection(name).find({ tenantId }).toArray()`
    - Store as a separate document in `tenant_deletion_snapshots`: `{ snapshotId, tenantId, collection: name, documents: docs, documentCount: docs.length, createdAt: new Date() }`
    - This avoids the 16MB BSON limit by splitting per collection
  - Also snapshot the tenant document itself: `db.collection('tenant').findOne({ _id: ObjectId.createFromHexString(tenantId) })`
  - Store tenant doc as `{ snapshotId, tenantId, collection: 'tenant', documents: [tenantDoc], documentCount: 1, createdAt: new Date() }`
  - Return `{ snapshotId, collectionCount, totalDocuments }`

  `async purgeTenant(tenantId, snapshotId)`:
  - Get db via `getDB()`
  - Start session: `const session = db.client.startSession()` (NOTE: use `db.client` not `db` for startSession)
  - Wait, check the existing pattern in cascadeDeletion.service.js -- it uses `db.startSession()`. Verify which works. Use whichever pattern the codebase already uses: `const session = db.startSession()`
  - Inside `session.withTransaction(async () => { ... })`:
    - For each collection in `TENANT_SCOPED_COLLECTIONS` (SEQUENTIALLY, not parallel -- MongoDB transactions don't support parallel ops):
      - `await db.collection(name).deleteMany({ tenantId }, { session })`
    - Delete the tenant document: `await db.collection('tenant').deleteOne({ _id: ObjectId.createFromHexString(tenantId) }, { session })`
  - After transaction completes, return `{ success: true, snapshotId, tenantId }`
  - In finally block: `await session.endSession()`
  - On error: throw with clear message

  **Add to `api/super-admin/super-admin.service.js`:**

  2. Import `tenantPurgeService` from `../../services/tenantPurge.service.js`
  3. Import `AUDIT_ACTIONS, COLLECTIONS` from `../../config/constants.js` (COLLECTIONS already imported)
  4. Import `AUDIT_ACTIONS` from constants (add to existing import)

  Add these methods to the `superAdminService` export:

  `async deletionPreview(tenantId)`:
  - Verify tenant exists (same pattern as getTenantWithStats)
  - Call `tenantPurgeService.previewDeletion(tenantIdString)` where `tenantIdString = tenant.tenantId || tenant._id.toString()`
  - Return preview result with tenant name

  `async softDeleteTenant(tenantId, { gracePeriodDays = 30, reason = '' } = {}, actorId)`:
  - Find tenant, verify exists and is not already in deletionStatus
  - Update tenant document with:
    ```
    {
      isActive: false,
      deletionStatus: 'scheduled',
      deletionScheduledAt: new Date(),
      deletionPurgeAt: new Date(Date.now() + gracePeriodDays * 24 * 60 * 60 * 1000),
      deletionRequestedBy: actorId,
      deletionReason: reason,
      updatedAt: new Date()
    }
    ```
  - Call `auditTrailService.logAction(AUDIT_ACTIONS.TENANT_SOFT_DELETED, actorId, { targetId: tenantId, tenantName: tenant.name, gracePeriodDays, reason })`
  - Return updated tenant

  `async cancelDeletion(tenantId, actorId)`:
  - Find tenant, verify exists and `deletionStatus === 'scheduled'`
  - If `deletionStatus` is not 'scheduled', throw error
  - Update tenant:
    ```
    {
      $set: { deletionStatus: 'cancelled', updatedAt: new Date() },
      $unset: { deletionPurgeAt: '', deletionRequestedBy: '', deletionReason: '' }
    }
    ```
  - Note: Keep `isActive: false` -- cancelling deletion does NOT reactivate. Super admin must explicitly toggle-active to reactivate.
  - Call `auditTrailService.logAction(AUDIT_ACTIONS.TENANT_DELETION_CANCELLED, actorId, { targetId: tenantId, tenantName: tenant.name })`
  - Return updated tenant

  `async purgeTenant(tenantId, actorId)`:
  - Find tenant, verify exists
  - Verify `deletionStatus === 'scheduled'` (must be soft-deleted first)
  - Update tenant `deletionStatus` to `'purging'`
  - Step 1: Create snapshot via `tenantPurgeService.createTenantSnapshot(tenantIdString)`
  - Step 2: Purge via `tenantPurgeService.purgeTenant(tenantIdString, snapshotResult.snapshotId)`
  - Step 3: Log audit via `auditTrailService.logAction(AUDIT_ACTIONS.TENANT_PURGED, actorId, { targetId: tenantId, tenantName: tenant.name, snapshotId })`
  - Return purge result
  - On error: update tenant `deletionStatus` back to `'scheduled'`, re-throw

  `async getPlatformAuditLog(filters)`:
  - Import and call `auditTrailService.getAuditLog(filters)`
  - Return results

  `async getTenantAuditLog(tenantId)`:
  - Call `auditTrailService.getAuditLogForTenant(tenantId)`
  - Return results
