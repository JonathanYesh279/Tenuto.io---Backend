  <action>
Add two items to `theory-course.service.js`:

1. **`_unionStudentIds(arrayOfArrays)` helper** (private, not exported):
   - Takes an array of arrays (each being a lesson's `studentIds`).
   - Returns a flat array of unique student ID strings using a Set.
   - Handles null/undefined inner arrays gracefully.

2. **`autoGroupLessons({ schoolYearId, minLessonCount = 2, dryRun = false }, options = {})` function**:
   - Extract `tenantId` from `options.context?.tenantId` via `requireTenantId`.
   - Run MongoDB aggregation pipeline on `theory_lesson` collection:
     - `$match`: `{ tenantId, courseId: null, schoolYearId }` — only ungrouped lessons in the given school year.
     - `$group` by `{ category: '$category', teacherId: '$teacherId', dayOfWeek: '$dayOfWeek', startTime: '$startTime', endTime: '$endTime' }`. Accumulate: `lessonIds: { $push: { $toString: '$_id' } }`, `allStudentIdArrays: { $push: '$studentIds' }`, `lessonCount: { $sum: 1 }`, `minDate: { $min: '$date' }`, `maxDate: { $max: '$date' }`, `location: { $first: '$location' }`.
     - `$match`: `{ lessonCount: { $gte: minLessonCount } }` — skip singletons.
   - If `dryRun === true`: return `{ dryRun: true, groupsFound: groups.length, groups: [...] }` where each group has `{ category, teacherId, dayOfWeek, startTime, endTime, lessonCount, uniqueStudents }`. Do NOT create any courses.
   - If `dryRun === false`: For each group, compute `unionStudentIds` via `_unionStudentIds`, call existing `createCourse()` with `{ category, teacherId, dayOfWeek, startTime, endTime, location, studentIds: unionStudentIds, schoolYearId, startDate: minDate, endDate: maxDate, excludeDates: [] }`, then call existing `linkLessonsToCourse(courseId, lessonIds, options)`. Collect results: `{ courseId, category, lessonsLinked, studentsEnrolled }`.
   - Return `{ coursesCreated: results.length, results }`.
   - Wrap in try/catch, log errors via logger.

Follow the exact same error handling pattern as other functions in the file (logger.error + throw).

**In `theory.controller.js`:**

1. Add `autoGroupLessons` controller function:
   - Extract `{ schoolYearId, minLessonCount, dryRun }` from `req.body`.
   - If `!schoolYearId`, return `res.status(400).json({ error: 'schoolYearId is required' })`.
   - Call `theoryCourseService.autoGroupLessons({ schoolYearId, minLessonCount, dryRun }, { context: req.context })`.
   - Return `res.json({ data: result })`.
   - Wrap in try/catch with `next(err)`.

2. Add `linkLessonsToCourse` controller function:
   - Extract `courseId` from `req.params.id`.
   - Extract `{ lessonIds }` from `req.body`.
   - If `!lessonIds || !Array.isArray(lessonIds)`, return `res.status(400).json({ error: 'lessonIds array is required' })`.
   - Call `theoryCourseService.linkLessonsToCourse(courseId, lessonIds, { context: req.context })`.
   - Return `res.json({ data: result })`.
   - Wrap in try/catch with `next(err)`.

3. Add both to the `theoryController` export object.

**In `theory.route.js`:**

Add these two routes in the course routes section (after existing course routes, but still BEFORE `/:id` routes):

```javascript
router.post('/courses/auto-group', requirePermission('theory', 'create'), theoryController.autoGroupLessons);
router.post('/courses/:id/link-lessons', requirePermission('theory', 'create'), theoryController.linkLessonsToCourse);
```

Place `auto-group` BEFORE `courses/:id` routes since Express would otherwise match "auto-group" as an `:id` param. Best position: right after the `router.get('/courses', ...)` line, before `router.get('/courses/:id', ...)`.

IMPORTANT: The `auto-group` route MUST be placed before any `/courses/:id` parameterized routes to avoid Express treating "auto-group" as an ID value (same principle as Phase 84-02 decision).
