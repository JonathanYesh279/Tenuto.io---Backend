Create `src/components/settings/PermissionMatrixEditor.tsx` with the following behavior:

**Props:**
- `rolePermissions: Record<string, Record<string, Record<string, string>>>` — the full permission map from GET /api/settings/roles
- `onPermissionsChanged: () => void` — callback to trigger parent re-fetch after save/reset

**Layout:**
1. **Role dropdown** at the top — a `<select>` populated with all 13 TEACHER_ROLES. Default to first non-admin role (e.g., 'מורה').
2. **Grid** below the dropdown:
   - Rows = domains (from PERMISSION_DOMAIN_LABELS — 9 domains)
   - Columns = actions (from PERMISSION_ACTIONS_BY_DOMAIN for each domain — varies per domain)
   - Column headers show ACTION_LABELS Hebrew translations
   - Row headers show PERMISSION_DOMAIN_LABELS Hebrew translations
3. **Grid cells** show the current scope value. Use distinct styling:
   - `all` = green badge/chip
   - `department` = blue badge/chip
   - `own` = amber/yellow badge/chip
   - Empty/no permission = gray dash or empty
4. **Scope cycling** — Clicking a cell cycles through: empty -> own -> department -> all -> empty. Each click updates local state.
5. **Locked domain rows** — For non-admin-tier roles, rows for `settings` and `roles` domains should:
   - Be visually grayed out with a lock icon (use LockSimple from @phosphor-icons/react)
   - Not respond to clicks (disabled)
   - Show no scope values (these domains are not available to non-admin roles)
6. **Admin-tier role lockout** — When an admin-tier role is selected in the dropdown:
   - ALL cells are locked/disabled (admin permissions are immutable)
   - Show a small info banner: "הרשאות הנהלה לא ניתנות לשינוי" (admin permissions cannot be changed)
   - The "Save" and "Reset" buttons should be hidden or disabled

**Save button:**
- Below the grid, a "Save" button. On click, call `rolesService.updateRolePermissions(selectedRole, permissionsObject)` where permissionsObject is the domains/actions/scopes for the selected role.
- The permissions object shape: `{ students: { view: "all", create: "own", ... }, ... }` — only include domains/actions that have a scope value (omit empty ones).
- Show toast on success ("ההרשאות עודכנו בהצלחה") or error.
- Disable while saving.

**Reset to Default button:**
- Next to Save, a "Reset to Default" button (secondary styling).
- Calls `rolesService.resetRolePermissions(selectedRole)`.
- On success, update local state with the returned default permissions and show toast "ההרשאות אופסו לברירת מחדל".
- Call `onPermissionsChanged()` after save or reset.

**Styling:**
- Use the same card styling (bg-white rounded-2xl border border-gray-200 p-5).
- Grid should use a proper HTML table or CSS grid. Use `border-collapse` and subtle borders.
- RTL-compatible (Hebrew labels).
- Scope badges should be compact (px-2 py-0.5 text-xs rounded-full).

**State management:**
- Local state for selectedRole and editedPermissions (cloned from rolePermissions[selectedRole]).
- When role dropdown changes, reload editedPermissions from props.
- Track dirty state (hasChanges) to enable/disable save button.
