In `tailwind.config.js`, add two new sections to `theme.extend`:

**1. Semantic typography scale (TYP-01):**

Add a `fontSize` key to `theme.extend`:
```javascript
fontSize: {
  // Semantic type scale — Hebrew-optimized line heights (TYP-01)
  'display': ['2.25rem', { lineHeight: '1.3', fontWeight: '700' }],   // 36px — hero numbers, big stats
  'h1': ['1.875rem', { lineHeight: '1.3', fontWeight: '700' }],       // 30px — page titles
  'h2': ['1.5rem', { lineHeight: '1.35', fontWeight: '600' }],        // 24px — section headings
  'h3': ['1.25rem', { lineHeight: '1.4', fontWeight: '600' }],        // 20px — card headings
  'body': ['0.875rem', { lineHeight: '1.6', fontWeight: '400' }],     // 14px — default body text
  'small': ['0.8125rem', { lineHeight: '1.5', fontWeight: '400' }],   // 13px — secondary text
  'caption': ['0.75rem', { lineHeight: '1.5', fontWeight: '400' }],   // 12px — metadata, labels
},
```

This produces Tailwind classes: `text-display`, `text-h1`, `text-h2`, `text-h3`, `text-body`, `text-small`, `text-caption`. Each class sets font-size, line-height, AND font-weight in one utility.

Hebrew line heights: headings use 1.3-1.4 (snug but not clipping), body uses 1.6 (relaxed for Hebrew readability). These are conservative values that avoid clipping Hebrew descenders.

Note: These ADD new fontSize entries. They do NOT override Tailwind's built-in `text-sm`, `text-lg`, etc. which continue to work as before.

**2. Table density tokens (TCS-01):**

There is no built-in Tailwind section for "table density" so define these as CSS custom properties. Add them to `theme.extend.spacing` (alongside the spacing tokens from Task 1):


These become usable as: `h-table-row-compact`, `px-table-cell-px`, `py-table-cell-py`, `py-table-header-py`.

**Important:** Do NOT modify any component files. Do NOT change existing fontSize or spacing entries. Only ADD new ones.
