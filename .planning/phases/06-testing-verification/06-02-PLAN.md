Create `test/tenant-isolation/read-isolation.test.js` with the following structure:

**Setup pattern (CRITICAL — ESM import order matters):**
```
1. import { describe, it, expect, beforeAll, afterAll, beforeEach } from 'vitest'
2. import { setupTenantIsolationDB, teardownTenantIsolationDB, resetCollections, patchMongoDBService } from './setup.js'
3. In beforeAll:
   a. Call setupTenantIsolationDB() to get { db }
   b. Call patchMongoDBService(db) to override mongoDB.service.js BEFORE any app imports
   c. Dynamically import fixtures: const { seedTwoTenants, ... } = await import('./fixtures/two-tenant-seed.js')
   d. Dynamically import test app: const { createTestApp } = await import('./helpers/test-app.js')
   e. Dynamically import token helper: const { makeToken } = await import('./helpers/token.js')
   f. Call seedTwoTenants(db)
   g. Create app = createTestApp()
   h. Create tokenA = makeToken(teacherA), tokenB = makeToken(teacherB)
4. In afterAll: call teardownTenantIsolationDB()
5. In beforeEach: call resetCollections(), then re-seed with seedTwoTenants(db)
```

**Test describe blocks:**

**`describe('Cross-Tenant Read Isolation — Student')`:**
- `it('GET /api/student — Tenant A sees only Tenant A students')`: Request with tokenA, expect 200, verify every returned student has `tenantId === TENANT_A_ID`. Verify studentB._id is NOT in results.
- `it('GET /api/student/:id — Tenant A fetching Tenant B student gets 404')`: Request with tokenA to `/api/student/${studentB._id}`, expect 404 status (NOT 403 — 404 prevents existence leaking).
- `it('GET /api/student — Tenant B sees only Tenant B students')`: Same test from Tenant B perspective for symmetry.

**`describe('Cross-Tenant Read Isolation — Teacher')`:**
- `it('GET /api/teacher — Tenant A sees only Tenant A teachers')`: Request with tokenA, verify all returned teachers have tenantId === TENANT_A_ID.
- `it('GET /api/teacher/:id — Tenant A fetching Tenant B teacher gets 404')`: Request with tokenA to `/api/teacher/${teacherB._id}`, expect 404.

**`describe('Cross-Tenant Read Isolation — Orchestra')`:**
- `it('GET /api/orchestra — Tenant A sees only Tenant A orchestras')`: Verify all results have TENANT_A_ID.
- `it('GET /api/orchestra/:id — cross-tenant fetch returns 404')`: Fetch orchestraB with tokenA, expect 404.

**`describe('Cross-Tenant Read Isolation — School Year')`:**
- `it('GET /api/school-year — Tenant A sees only Tenant A school years')`: Verify results scoped.
- `it('GET /api/school-year/:id — cross-tenant fetch returns 404')`: Fetch schoolYearB with tokenA.

**Important test patterns:**
- Use `supertest` (import request from 'supertest') for HTTP requests.
- Set `Authorization: Bearer ${tokenA}` header on every request.
- For list endpoints: assert `res.status === 200`, iterate results, assert every item has correct tenantId, assert opposite tenant's item is absent.
- For getById endpoints: assert `res.status === 404` (per Phase 5 IDOR-safe error handling).
- Do NOT mock any middleware. The entire point is that real enforceTenant and real buildScopedFilter run.
- Store all dynamic imports in module-level `let` variables so test blocks can access them.
