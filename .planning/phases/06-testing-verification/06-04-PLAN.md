**.github/workflows/tenant-isolation.yml** — CI pipeline for tenant isolation tests:

```yaml
name: Tenant Isolation Tests

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  tenant-isolation:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [20.x]

    steps:
      - uses: actions/checkout@v4

      - name: Use Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'

      - name: Cache MongoDB Memory Server binary
        uses: actions/cache@v4
        with:
          path: node_modules/.cache/mongodb-memory-server
          key: mms-binary-${{ runner.os }}-6.0.0

      - name: Install dependencies
        run: npm ci

      - name: Run route accountability check
        run: node utils/validateAllowlist.js

      - name: Run tenant isolation tests
        run: npm run test:tenant-isolation
        env:
          NODE_ENV: test
          ACCESS_TOKEN_SECRET: test-access-secret
          REFRESH_TOKEN_SECRET: test-refresh-secret
          LOG_LEVEL: silent
```

Key decisions:
- Cache MMS binary to avoid 30-60s download on each run (binary stored in `node_modules/.cache/mongodb-memory-server`).
- Run route accountability check FIRST (fast, catches obvious gaps before slow integration tests).
- Run tenant isolation test suite (includes read, write, middleware, concurrent, allowlist, route-accountability tests).
- Node 20.x matches the project's current Node version.
- Do NOT create the `.github/workflows` directory until confirming it doesn't exist (already verified: it doesn't).
