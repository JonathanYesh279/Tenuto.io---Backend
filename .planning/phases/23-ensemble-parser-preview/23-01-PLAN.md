  <name>Task 1: Ensemble helper functions and constants</name>
  <files>api/import/import.service.js, config/constants.js</files>
  <action>
  Add the following helper functions and constants to `api/import/import.service.js`, placed after the existing `isColoredCell()` function (around line 274) in a new section `// --- Ensemble Import Helpers ---`:

  1. **`excelTimeToHHMM(value)`** — Converts Excel time values to "HH:MM" strings. Handle three cases:
     - String already in "HH:MM" format: extract and zero-pad
     - Date object: extract UTC hours and minutes (ExcelJS may return Date for time cells)
     - Numeric serial (0-1 range): convert using `Math.round(value * 24 * 60)` for total minutes, then derive hours/minutes
     - Return null for null/undefined/unrecognizable values

  2. **`HEBREW_DAY_MAP`** constant — Maps Hebrew day names and abbreviations to dayOfWeek numbers:
     - 'ראשון'/`'יום ראשון'`/`"א'"` -> 0, 'שני'/`'יום שני'`/`"ב'"` -> 1, through 'שישי'/`"ו'"` -> 5, 'שבת'/`'יום שבת'` -> 6
     - Also include without the `'יום '` prefix

  3. **`hebrewDayToNumber(dayText)`** — Looks up `dayText.trim()` in `HEBREW_DAY_MAP`, returns the number or null if not found.

  4. **`SUBTYPE_KEYWORDS`** array — Ordered list of `{ keyword, subType }` objects for matching within ensemble names. Keywords (in detection order, longest first to avoid partial matches):
     - 'קאמרי קלאסי' -> 'קאמרי קלאסי'
     - 'כלי נשיפה' -> 'כלי נשיפה'
     - 'כלי קשת' -> 'כלי קשת'
     - 'סימפונית' -> 'סימפונית'
     - 'ביג-בנד' -> 'ביג-בנד'
     - "ג'אז" -> "ג'אז-פופ-רוק"
     - 'פופ' -> "ג'אז-פופ-רוק"
     - 'רוק' -> "ג'אז-פופ-רוק"
     - 'עממית' -> 'עממית'
     - 'מקהלה' -> 'מקהלה'
     - 'קולי' -> 'קולי'
     - 'קאמרי' -> 'קאמרי קלאסי' (abbreviated form, MUST come after 'קאמרי קלאסי')

  5. **`decomposeEnsembleName(rawName, participantCount)`** — Returns `{ name, type, subType, warning }`:
     - Store `rawName.trim()` as `name` verbatim (per locked decision)
     - Detect type from prefix: `/^תז[׳'`]/` or `startsWith('תזמורת')` -> 'תזמורת'; `startsWith('הרכב')` -> 'הרכב'; `startsWith('מקהלה')` -> 'הרכב' (choir is ensemble type, subType will be מקהלה)
     - If no prefix match: fallback to participant count: `participantCount > 12` -> 'תזמורת', else 'הרכב'
     - Detect subType by iterating `SUBTYPE_KEYWORDS` and checking `name.includes(keyword)` — first match wins
     - If subType is null after iteration, set `warning: 'לא ניתן לזהות סוג משנה מהשם'`
     - Otherwise warning is null

  6. **`detectPerformanceLevel(cells, perfLevelCols)`** — Takes an array of ExcelJS cell objects and an object `{ beginnerCol, intermediateCol, representativeCol }` with column indices. Checks each of the three cells using the existing `isColoredCell(cell.fill)`. Returns the matching level string ('התחלתי'/'ביניים'/'ייצוגי') or null if none are colored.

  In `config/constants.js`: Add `'ensembles'` to the `IMPORT_TYPES` array (currently `['teachers', 'students']` — update to `['teachers', 'students', 'ensembles']`).

<task type="auto">
  <name>Task 2: Ensemble sheet parser with fixed-position column parsing and analytics extraction</name>
  <files>api/import/import.service.js</files>
  <action>
  Add `parseEnsembleSheet(buffer)` as an async function in `api/import/import.service.js`, placed after the helper functions from Task 1. This is the main parser function. Follow the `parseMinistryTeacherSheet()` pattern for ExcelJS workbook loading and row iteration. Use FIXED-POSITION column parsing (not header detection) per locked decision about duplicate Activity I/II column names.

  **Sheet discovery:**
  ```javascript
  const ENSEMBLE_SHEET_NAME = 'הרכבי ביצוע';
  const worksheet = workbook.worksheets.find(ws => ws.name === ENSEMBLE_SHEET_NAME);
  if (!worksheet) return null;
  ```

  **Row collection:** Use the same pattern as `parseMinistryTeacherSheet()` — iterate with `worksheet.eachRow({ includeEmpty: false })`, collect `{ rowNumber, cells, texts }` entries. Use the same text extraction logic (richText, formula results, Date filtering, Unicode cleanup).

  **Header detection:** Scan first 10 rows for the header row. The header row is identified by finding the row that contains the text 'שם המנצח/מדריך' OR 'מספר משתתפים' (conductor name or participant count column header). Use a scoring approach: for each candidate row, count matches against known header fragments: 'מנצח', 'מדריך', 'משתתפים', 'נגנים', 'ביום', 'משעה', 'עד שעה', 'שעות בפועל'. Row with the highest score (min 3) is the header.

  **Fixed-position column mapping:** Once the header row is found, scan its texts to map column indices. Map column indices by finding specific text patterns in the header:
  - `conductorCol`: text containing 'מנצח' or 'מדריך'
  - `nameCol`: text containing 'תזמורת' and ('הרכב' or 'כלי' or 'קולי')
  - `participantCol`: text containing 'משתתפים' or 'נגנים' or 'זמרים'
  - Activity 1 sub-columns: find FIRST occurrence of 'ביום', then the next 3 columns are 'משעה', 'עד שעה', 'שעות בפועל' — store as `act1DayCol, act1StartCol, act1EndCol, act1HoursCol`
  - Activity 2 sub-columns: find SECOND occurrence of 'ביום' (after Activity 1), then same pattern — store as `act2DayCol, act2StartCol, act2EndCol, act2HoursCol`. If no second occurrence, set all act2 columns to null.
  - `totalWeeklyHoursCol`: text matching 'ש"ש' and ('הרכב' or 'ביצוע' or 'בפועל')
  - `coordHoursCol`: text containing 'הכנה' or 'ריכוז'
  - `totalReportingCol`: text containing 'דיווח' or 'ימ"ש'
  - Performance level columns: find 3 consecutive columns containing 'התחלתי', 'ביניים', 'ייצוגי' (scan parent rows above the header too, as Ministry files have multi-row merged headers). Store as `perfBeginnerCol, perfIntermediateCol, perfRepresentativeCol`.
  - `ministryUseCol`: text containing 'לשימוש המשרד' or just 'לשימוש'

  If critical columns are missing (conductor, name, participant count): return `{ error: 'missing_columns', details: [...] }`.

  Also check parent rows above header row for performance level column text (multi-row merged headers). Scan rows from headerIdx-1 up to headerIdx-3 for 'התחלתי'/'ביניים'/'ייצוגי' at columns that were not found in the main header row.

  **Data row parsing:** For each row after the header:
  - Extract conductor name from `conductorCol` (text value)
  - Extract ensemble name from `nameCol` (text value)
  - Extract participant count from `participantCol` (numeric value, default to 0)
  - **Boundary detection:** Stop parsing data rows when:
    - The conductor column AND name column are both empty for a row, AND
    - The row is not just a gap (check if the NEXT row also has empty conductor+name — two consecutive empty rows signal end of data)
    - OR the text in any of the first 3 columns contains a known analytics section header keyword: 'סיכום', 'סך גופי', 'תזמורות כלי', 'סך משתתפים'
  - Call `decomposeEnsembleName(rawName, participantCount)` for name decomposition
  - Call `detectPerformanceLevel(cells, perfLevelCols)` for performance level
  - Build schedule from Activity 1 and Activity 2 columns:
    ```javascript
    const activity1 = {
      day: texts[act1DayCol] || null,
      dayOfWeek: hebrewDayToNumber(texts[act1DayCol]),
      startTime: excelTimeToHHMM(cells[act1StartCol]?.value),
      endTime: excelTimeToHHMM(cells[act1EndCol]?.value),
      actualHours: parseFloat(texts[act1HoursCol]) || null,
    };
    // Activity 2 only if act2 columns exist and have data
    ```
  - Extract hours: `totalActual` from totalWeeklyHoursCol, `coordinationHours` from coordHoursCol, `totalReporting` from totalReportingCol — all as `parseFloat(text) || null`
  - Extract `ministryUseCode` from ministryUseCol as `parseInt(text) || null`

  Each parsed row produces:
  ```javascript
  {
    row: rowNumber,                    // Excel row number
    rawName: ensembleName,             // Verbatim from file
    ...decomposition,                  // { name, type, subType, warning }
    performanceLevel: detectedLevel,   // from cell color or null
    participantCount: number,
    conductorName: conductorText,      // Raw text for matching later
    schedule: { activity1, activity2: activity2 || null },
    hours: { totalActual, coordinationHours, totalReporting },
    ministryUseCode,
  }
  ```

  **Analytics mini-table parsing:** After the data boundary is detected, continue scanning remaining rows. Build `parseAnalyticsMiniTable(rowEntries, startIdx)` function:
  - Look for rows containing known category names: 'תזמורות כלי נשיפה', 'תזמורות סימפוניות', 'תזמורות כלי קשת', 'הרכבים קאמריים', 'מקהלות', etc.
  - For each category row, try to extract: category name (first text cell), ensemble count (next numeric cell), participant total (next numeric cell)
  - Return `{ byType: [{ category, ensembleCount, participantCount }], totalEnsembles, totalParticipants, performanceLevelBreakdown: { beginner, intermediate, representative } }` or null if no analytics section found
  - This is best-effort parsing — if the format is unrecognizable, return null and add no warnings (analytics is informational, not critical)

  **Return value:**
  ```javascript
  return {
    parsedRows: [...],     // Array of parsed ensemble objects
    analytics: miniTable,  // Analytics object or null
    warnings: [...],       // Array of { row, field, message } from name decomposition + missing data
  };
  ```

  Warnings to generate per row:
  - `decomposeEnsembleName` returned a warning -> add `{ row, field: 'name', message: warning }`
  - `performanceLevel` is null -> add `{ row, field: 'performanceLevel', message: 'לא זוהתה רמת ביצוע (אין צבע רקע)' }`
  - `participantCount` is 0 or null -> add `{ row, field: 'participantCount', message: 'מספר משתתפים חסר' }`
  - conductor name is empty -> add `{ row, field: 'conductor', message: 'שם מנצח/מדריך חסר' }`
  </action>
