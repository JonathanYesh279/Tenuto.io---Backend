  <files>api/reports/generators/student-attendance.generator.js</files>
  <action>
Create `student-attendance.generator.js` following the established generator plugin convention.

**Generator metadata:**
- id: 'student-attendance'
- name: 'נוכחות תלמידים' (Student Attendance)
- description: 'אחוזי נוכחות לתלמיד עם מגמות לפי סוג פעילות'
- category: 'student'
- icon: 'ChartBar'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - activityType: { type: 'string', required: false, default: 'all', allowed: ['all', 'שיעור פרטי', 'תאוריה', 'חזרות', 'תזמורת'] }
  - department: { type: 'string', required: false }
- exports: ['excel', 'pdf']

**Columns:**
- studentName (string, sortable) -- 'שם תלמיד/ה'
- class (string, sortable) -- 'כיתה'
- primaryInstrument (string, sortable) -- 'כלי נגינה'
- totalLessons (number, sortable) -- 'סה"כ שיעורים'
- attended (number, sortable) -- 'נוכח/ת'
- missed (number, sortable) -- 'חיסורים'
- attendanceRate (percentage, sortable) -- 'אחוז נוכחות'
- trend (string, sortable) -- 'מגמה'

**generate(params, scope, { services }) logic:**

NOTE: Do NOT use the attendanceAnalyticsService directly (it's designed for single-student lookups). Instead, query activity_attendance collection directly for bulk aggregation, following the same pattern the analytics service uses internally.

1. Build student filter: `{ tenantId: scope.tenantId, isActive: true }`.
2. Apply department filtering (same pattern as other student generators using getInstrumentsByDepartment).
3. Apply scope filtering:
   - scope.type === 'department': instrument-based filtering
   - scope.type === 'own': filter by teacherAssignments.teacherId
4. Query student collection to get student list (project: _id, personalInfo, academicInfo).
5. Build activity_attendance filter: `{ tenantId: scope.tenantId, studentId: { $in: studentIdStrings } }`. If params.activityType !== 'all', add `activityType: params.activityType`.
6. Query activity_attendance collection with filter.
7. Group attendance records by studentId in-memory:
   - For each student, count total records, count status === 'הגיע/ה' (attended), count status === 'לא הגיע/ה' (missed).
   - Calculate attendanceRate = total > 0 ? round(attended / total * 100, 2) : 0
   - Calculate trend: take last 10 records sorted by date desc, split into recent 5 and older 5. If recent rate > older rate + 10% => 'שיפור' (improving), if < -10% => 'ירידה' (declining), else 'יציב' (stable). If fewer than 6 records => 'אין מספיק נתונים'.
8. Map to rows, including students with zero attendance records (totalLessons=0, attendanceRate=0, trend='אין מספיק נתונים').
9. Summary items:
   - total students (number) -- 'סה"כ תלמידים'
   - average attendance rate (number) -- 'ממוצע נוכחות %'
   - students below 70% (number) -- 'תלמידים מתחת ל-70%'
   - students above 90% (number) -- 'תלמידים מעל 90%'
10. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** The activity_attendance collection stores records with fields: studentId (string), teacherId (string), activityType (string, Hebrew), date (Date), status (string, Hebrew: 'הגיע/ה', 'לא הגיע/ה', 'cancelled'), tenantId (string). Query it directly, do not import the attendance analytics service.

<task type="auto">
  <name>Task 2: Create Orchestra Participation generator (STUD-04)</name>
  <files>api/reports/generators/orchestra-participation.generator.js</files>
  <action>
Create `orchestra-participation.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'orchestra-participation'
- name: 'השתתפות בתזמורות' (Orchestra Participation)
- description: 'רישום תלמידים בתזמורות וזיהוי חפיפה בין הרכבים'
- category: 'student'
- icon: 'MusicNotes'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - orchestraId: { type: 'string', required: false }
  - overlapOnly: { type: 'string', required: false, default: 'false', allowed: ['true', 'false'] }
- exports: ['excel', 'pdf']

**Columns:**
- studentName (string, sortable) -- 'שם תלמיד/ה'
- class (string, sortable) -- 'כיתה'
- primaryInstrument (string, sortable) -- 'כלי נגינה'
- orchestraNames (string) -- 'תזמורות'
- orchestraCount (number, sortable) -- 'מספר תזמורות'
- hasOverlap (string, sortable) -- 'חפיפה'

**generate(params, scope, { services }) logic:**
1. Query orchestra collection: `{ tenantId: scope.tenantId }`. If params.orchestraId, filter to that specific orchestra. For scope.type === 'own', this is a student report so use scope filtering on student side.
2. Build a map: studentId -> [orchestraName, ...] from each orchestra's `memberIds` array. Each orchestra has `memberIds` (array of student ID strings) and `name` field.
3. Query student collection: `{ tenantId: scope.tenantId, isActive: true }`. Apply department filtering and scope filtering (same as other student generators).
4. For each student:
   - Look up which orchestras they belong to using the student-to-orchestras map (match student._id.toString() against memberIds).
   - studentName = `${student.personalInfo?.lastName || ''} ${student.personalInfo?.firstName || ''}`.trim()
   - class = student.academicInfo?.class || ''
   - primaryInstrument: primary instrument from instrumentProgress
   - orchestraNames: joined orchestra names (comma-separated), or '-' if none
   - orchestraCount: number of orchestras
   - hasOverlap: orchestraCount > 1 ? 'כן' : 'לא'
5. If params.overlapOnly === 'true', filter to only students with orchestraCount > 1.
6. If params.orchestraId is set, filter to only students who are members of that specific orchestra (they may also appear in the map for other orchestras -- show all their orchestras in the orchestraNames field for context).
7. Summary items:
   - total students in orchestras (number) -- 'תלמידים בתזמורות'
   - total orchestras (number) -- 'סה"כ תזמורות'
   - students in multiple orchestras (number) -- 'תלמידים עם חפיפה'
   - average orchestras per student (number) -- 'ממוצע תזמורות לתלמיד'
   - students not in any orchestra (number) -- 'תלמידים ללא תזמורת'
8. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Orchestra documents have `memberIds` as an array of string IDs (not ObjectIds). The orchestra `name` field stores the display name. Import `getInstrumentDepartment`, `getInstrumentsByDepartment` from constants for department filtering.
  </action>
  <verify>
Call `GET /api/reports/registry` -- should include 'orchestra-participation'. Call `GET /api/reports/orchestra-participation` with auth -- should return rows with orchestra membership info. Call with `?overlapOnly=true` -- should return only students in multiple orchestras.
  </verify>
  <done>Orchestra Participation generator returns per-student orchestra membership, identifies overlap (students in multiple orchestras), and summary shows enrollment distribution.</done>
