  <name>Task 1: Create Student Enrollment Status generator (STUD-01)</name>
  <files>api/reports/generators/student-enrollment.generator.js</files>
  <action>
Create `student-enrollment.generator.js` following the established generator plugin convention (see `_example.generator.js` and `teacher-roster.generator.js` for the pattern).

**Generator metadata:**
- id: 'student-enrollment'
- name: 'סטטוס רישום תלמידים' (Student Enrollment Status)
- description: 'סטטוס פעיל/לא פעיל עם פילוח לפי כלי נגינה וכיתה'
- category: 'student'
- icon: 'Users'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - status: { type: 'string', required: false, default: 'all', allowed: ['all', 'active', 'inactive'] }
  - department: { type: 'string', required: false }
  - class: { type: 'string', required: false }
- exports: ['excel', 'pdf']

**Columns:**
- studentName (string, sortable) -- 'שם תלמיד/ה'
- class (string, sortable) -- 'כיתה'
- primaryInstrument (string, sortable) -- 'כלי נגינה ראשי'
- department (string, sortable) -- 'מחלקה'
- studyYears (number, sortable) -- 'שנות לימוד'
- currentStage (number, sortable) -- 'שלב נוכחי'
- isActive (string, sortable) -- 'סטטוס'

**generate(params, scope, { services }) logic:**
1. Build filter: `{ tenantId: scope.tenantId }`.
2. Status filter: if params.status === 'active' add `isActive: true`, if 'inactive' add `isActive: false`, if 'all' no isActive filter.
3. Class filter: if params.class, add `{ 'academicInfo.class': params.class }`.
4. Department filter (params.department): use `getInstrumentsByDepartment(params.department)` from constants.js. If instruments found, add `{ 'academicInfo.instrumentProgress.instrumentName': { $in: instruments } }`. If no instruments, return emptyResult.
5. Scope-based filtering:
   - scope.type === 'department': get all instruments from scope.departmentIds via getInstrumentsByDepartment, intersect with any department param filter, add to filter.
   - scope.type === 'own': this is a student report, so for 'own' scope (teacher), query student collection for students whose `teacherAssignments.teacherId` matches scope.teacherId.
6. Query student collection with built filter, sort by `personalInfo.lastName`.
7. Map each student to a row:
   - studentName = `${student.personalInfo?.lastName || ''} ${student.personalInfo?.firstName || ''}`.trim()
   - class = student.academicInfo?.class || ''
   - primaryInstrument: find entry in `academicInfo.instrumentProgress` where `isPrimary === true`, use its `instrumentName`. Fallback to first entry if no primary.
   - department: use `getInstrumentDepartment(primaryInstrument)` from constants.js
   - studyYears = student.academicInfo?.studyYears || 0
   - currentStage: from the primary instrument progress entry, use `currentStage` field. Default 0.
   - isActive = student.isActive ? 'פעיל/ה' : 'לא פעיל/ה'
8. Summary items:
   - total students count (number) -- 'סה"כ תלמידים'
   - active count (number) -- 'פעילים'
   - inactive count (number) -- 'לא פעילים'
   - distinct instruments count (number) -- 'כלי נגינה שונים'
   - distinct classes count (number) -- 'כיתות שונות'
9. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Import `getInstrumentDepartment` and `getInstrumentsByDepartment` from `../../../config/constants.js` (ESM, 3 levels up from generators/).
  </action>
  <verify>
Start the server and call: `curl -s http://localhost:PORT/api/reports/registry | jq '.[] | select(.id == "student-enrollment")'` -- should return the generator metadata. Then call `GET /api/reports/student-enrollment` with auth -- should return valid { metadata, columns, rows, summary } response.
  </verify>
  <done>Student Enrollment Status generator is auto-discovered by registry, returns per-student rows with instrument/class/stage info, and summary with active/inactive counts and distribution metrics.</done>

<task type="auto">
  <name>Task 2: Create Student-Teacher Assignments generator (STUD-03)</name>
  <files>api/reports/generators/student-teacher-assignments.generator.js</files>
  <action>
Create `student-teacher-assignments.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'student-teacher-assignments'
- name: 'שיבוצי מורים-תלמידים' (Student-Teacher Assignments)
- description: 'מי לומד אצל מי וזיהוי תלמידים ללא שיבוץ'
- category: 'student'
- icon: 'UserSwitch'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - assignmentStatus: { type: 'string', required: false, default: 'all', allowed: ['all', 'assigned', 'unassigned'] }
  - department: { type: 'string', required: false }
- exports: ['excel', 'pdf']

**Columns:**
- studentName (string, sortable) -- 'שם תלמיד/ה'
- class (string, sortable) -- 'כיתה'
- primaryInstrument (string, sortable) -- 'כלי נגינה'
- teacherName (string, sortable) -- 'שם מורה'
- day (string) -- 'יום'
- time (string) -- 'שעה'
- duration (number) -- 'משך (דקות)'
- assignmentStatus (string, sortable) -- 'סטטוס שיבוץ'

**generate(params, scope, { services }) logic:**
1. Build student filter: `{ tenantId: scope.tenantId, isActive: true }` (only active students for assignment tracking).
2. Department filter: same as enrollment generator -- filter by instrumentProgress.instrumentName using getInstrumentsByDepartment.
3. Scope-based filtering:
   - scope.type === 'department': intersect instrument filter with scope departments.
   - scope.type === 'own': filter students by `teacherAssignments.teacherId` === scope.teacherId.
4. Query student collection.
5. Collect all unique teacherIds from all students' teacherAssignments arrays.
6. Bulk-fetch those teachers from teacher collection: `{ _id: { $in: teacherObjectIds }, tenantId: scope.tenantId }`, project `{ personalInfo.firstName, personalInfo.lastName }`.
7. Build teacherLookup map: teacherId -> "lastName firstName".
8. For each student, produce one row per active teacherAssignment entry (a student with 2 teachers = 2 rows). If a student has zero active teacherAssignments, produce one row with teacherName = '-', day/time/duration empty, assignmentStatus = 'ללא שיבוץ'.
   - studentName = `${student.personalInfo?.lastName || ''} ${student.personalInfo?.firstName || ''}`.trim()
   - class = student.academicInfo?.class || ''
   - primaryInstrument: primary instrument from instrumentProgress (same logic as enrollment generator)
   - teacherName: from teacherLookup, or '-' if unassigned
   - day: assignment.day || ''
   - time: assignment.time || ''
   - duration: assignment.duration || 0
   - assignmentStatus: 'משובץ' for assigned, 'ללא שיבוץ' for unassigned
9. If params.assignmentStatus === 'assigned', filter rows to only 'משובץ'. If 'unassigned', filter to only 'ללא שיבוץ'.
10. Summary items:
   - total unique students (number) -- 'סה"כ תלמידים'
   - assigned students count (number) -- 'תלמידים משובצים'
   - unassigned students count (number) -- 'תלמידים ללא שיבוץ'
   - total assignments count (number) -- 'סה"כ שיבוצים'
   - distinct teachers count (number) -- 'מורים פעילים'
11. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Use `ObjectId` from 'mongodb' for teacher ID conversion. Import `getInstrumentDepartment` and `getInstrumentsByDepartment` from `../../../config/constants.js`. Only include `isActive: true` assignments in the output (filter assignment.isActive !== false).
  </action>
  <verify>
Call `GET /api/reports/registry` -- should include 'student-teacher-assignments'. Call `GET /api/reports/student-teacher-assignments` with auth -- should return rows with teacher names resolved. Call with `?assignmentStatus=unassigned` -- should return only students without active assignments.
  </verify>
  <done>Student-Teacher Assignments generator returns per-assignment rows linking students to teachers, identifies unassigned students, and summary shows assigned vs unassigned counts.</done>
