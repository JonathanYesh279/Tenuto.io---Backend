  <action>
1. In `super-admin.validation.js`, add a new `createTenantWithAdminSchema` Joi schema that validates the combined payload:
   - Tenant fields: `name` (string, required, trimmed), `slug` (string, required, trimmed, lowercase, pattern /^[a-z0-9-]+$/), `city` (string, required, trimmed)
   - Admin fields: `adminFirstName` (string, required, trimmed), `adminLastName` (string, required, trimmed), `adminEmail` (string, email, required)
   - Export the schema.

2. In `super-admin.service.js`, add a new `createTenantWithAdmin` function that:
   a. Validates input with `createTenantWithAdminSchema`.
   b. Imports `withTransaction` from `../../services/mongoDB.service.js`, `getDB` from `../../services/mongoDB.service.js`, `DEFAULT_ROLE_PERMISSIONS` from `../../config/permissions.js`, `bcrypt` (already imported), and `DEFAULT_PASSWORD` from `../../services/invitationConfig.js`.
   c. Uses `withTransaction(async (session) => { ... })` for atomicity.
   d. Inside the transaction:
      - Gets the db via `getDB()`.
      - Checks slug uniqueness: `db.collection('tenants').findOne({ slug: value.slug }, { session })`. If exists, throw `Tenant with slug "${value.slug}" already exists`.
      - Checks email uniqueness within the new tenant (not globally -- email is compound-indexed with tenantId). Since the tenant doesn't exist yet, just check across all tenants for safety: `db.collection('teachers').findOne({ 'credentials.email': value.adminEmail, isActive: true }, { session })` -- actually, email uniqueness is scoped by tenantId, so no cross-tenant check needed. Skip this check since the tenant is brand new (no existing teachers).
      - Creates the tenant document:
        ```js
        const tenantDoc = {
          name: value.name,
          slug: value.slug,
          city: value.city,
          director: { name: null, teacherId: null },
          ministryInfo: { institutionCode: null, districtName: null },
          settings: { lessonDurations: [30, 45, 60], schoolStartMonth: 9, rooms: [] },
          rolePermissions: { ...DEFAULT_ROLE_PERMISSIONS },
          subscription: { plan: 'basic', startDate: new Date(), endDate: null, isActive: true, maxTeachers: 50, maxStudents: 500 },
          conservatoryProfile: {},
          isActive: true,
          createdAt: new Date(),
          updatedAt: new Date(),
        };
        ```
        Note: `rolePermissions` must be a deep clone of DEFAULT_ROLE_PERMISSIONS because the original is frozen. Use a simple JSON.parse(JSON.stringify(DEFAULT_ROLE_PERMISSIONS)) to deep-clone since all values are strings (no dates/ObjectIds).
      - Inserts tenant: `const tenantResult = await db.collection('tenants').insertOne(tenantDoc, { session })`.
      - Gets the tenantId string: `const tenantId = tenantResult.insertedId.toString()`.
      - Hashes default password: `const hashedPassword = await bcrypt.hash(DEFAULT_PASSWORD, 10)`.
      - Creates the admin teacher document:
        ```js
        const adminTeacherDoc = {
          tenantId,
          personalInfo: {
            firstName: value.adminFirstName,
            lastName: value.adminLastName,
            email: value.adminEmail,
            phone: '',
            address: '',
            instrument: '',
            idNumber: '',
          },
          credentials: {
            email: value.adminEmail,
            password: hashedPassword,
            requiresPasswordChange: true,
            tokenVersion: 0,
            refreshToken: null,
            invitationMode: 'DEFAULT_PASSWORD',
            passwordSetAt: new Date(),
          },
          roles: ['מנהל'],
          teaching: { timeBlocks: [] },
          conducting: { orchestraIds: [] },
          ensemblesIds: [],
          schoolYears: [],
          isActive: true,
          createdAt: new Date(),
          updatedAt: new Date(),
        };
        ```
      - Inserts teacher: `await db.collection('teachers').insertOne(adminTeacherDoc, { session })`.
      - Returns `{ tenant: { _id: tenantResult.insertedId, ...tenantDoc }, adminTeacher: { _id: adminTeacherDoc._id, email: value.adminEmail, firstName: value.adminFirstName, lastName: value.adminLastName } }`.
   e. Export `createTenantWithAdmin` in the service object.
   f. Log tenant + admin creation: `log.info({ tenantId, adminEmail: value.adminEmail }, 'Tenant created with admin')`.

IMPORTANT: Use `COLLECTIONS.TENANT` and `COLLECTIONS.TEACHER` from constants.js for collection names (not hardcoded strings). Access them via `db.collection(COLLECTIONS.TENANT)`.
  </action>
