<task type="auto">
  <name>Task 1: Fix headerColMap passthrough in student import path (BUGF-01)</name>
  <files>api/import/import.service.js</files>
  <action>
Two one-line fixes in the `previewStudentImport` function:

1. **Line 1491** — Pass `headerColMap` to `detectInstrumentColumns`:
   Change: `const instrumentColumns = detectInstrumentColumns(headers);`
   To: `const instrumentColumns = detectInstrumentColumns(headers, headerColMap);`

2. **Line 1511** — Pass `headerColMap` to `mapColumns`:
   Change: `const mapped = mapColumns(row, STUDENT_COLUMN_MAP);`
   To: `const mapped = mapColumns(row, STUDENT_COLUMN_MAP, headerColMap);`

These two calls are the ONLY places where `headerColMap` is not passed in the student path. The teacher import path (`previewTeacherImport`) already passes `headerColMap` to both functions correctly — this brings student import to parity.

Context: `detectInstrumentColumns` uses `headerColMap` to find the column index of each header, which determines `instrumentSectionStart`. Without it, `colIndex` is always `undefined`, `instrumentSectionStart` stays `Infinity`, and the function returns an empty array — so zero instrument columns are ever detected for student imports.

Similarly, `mapColumns` uses `headerColMap` to disambiguate hours columns from role boolean columns via the `HOURS_FIELDS_WITH_ROLE_COLLISION` check. Without it, the collision guard is bypassed.
  </action>

1. `grep -n "detectInstrumentColumns(headers, headerColMap)" api/import/import.service.js` — confirms the fix on the correct line
2. `grep -n "mapColumns(row, STUDENT_COLUMN_MAP, headerColMap)" api/import/import.service.js` — confirms the fix on the correct line
3. `node -e "import('./api/import/import.service.js').then(() => console.log('OK')).catch(e => console.error(e))"` — confirms no syntax errors
  </verify>
  <done>
Both `detectInstrumentColumns` and `mapColumns` receive `headerColMap` in the student import path, achieving parity with teacher import. Student import preview now detects instrument columns correctly.
  </done>

**Part A — Extend STUDENT_COLUMN_MAP** (in `api/import/import.service.js`):

NOTE: `'מגמה'` already exists in `ROLE_COLUMN_NAMES` (teacher import, maps to role `'מגמה'`). This is NOT a collision because `STUDENT_COLUMN_MAP` and `ROLE_COLUMN_NAMES` are used in completely separate import paths (student vs teacher). The same Hebrew header legitimately means different things in student vs teacher contexts.

**Part B — Parse isBagrutCandidate as boolean** (in `api/import/import.service.js`):

Add truthy-value parsing in `validateStudentRow` function (after the age validation block, before the `return { errors, warnings }` on line ~1074):

```javascript
// Convert isBagrutCandidate to boolean
if (mapped.isBagrutCandidate !== undefined && mapped.isBagrutCandidate !== '') {
  mapped.isBagrutCandidate = TRUTHY_VALUES.includes(mapped.isBagrutCandidate);
} else {
  mapped.isBagrutCandidate = null;
}
```

This follows the same pattern as `mapped.extraHour` truthy-value parsing on lines 1015-1017. The `TRUTHY_VALUES` array (line 156) already handles all variants: checkmarks, V/v/x/X, 1, כן, true.

**Part C — Add isBagrutCandidate to Joi schemas** (in `api/student/student.validation.js`):

1. In `studentSchema` > `academicInfo` object (after line 120 `tests` field):
   ```javascript
   isBagrutCandidate: Joi.boolean().allow(null).default(null),
   ```

2. In `studentUpdateSchema` > `academicInfo` object (after line 201 `tests` field):
   ```javascript
   isBagrutCandidate: Joi.boolean().allow(null),
   ```

The field is optional (`.allow(null).default(null)`) so existing student documents without it will not break validation. This is the same backward-compatible pattern used by other nullable fields like `studyYears` and `extraHour`.
