<task type="auto">
  <name>Task 2: Create Teacher Workload Distribution generator (TCHR-02)</name>
  <files>api/reports/generators/teacher-workload.generator.js</files>
  <action>
Create `teacher-workload.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'teacher-workload'
- name: 'התפלגות עומס עבודה' (Workload Distribution)
- description: 'השוואת עומס עבודה בין מורים וזיהוי עומס יתר/חסר'
- category: 'teacher'
- icon: 'ChartBar'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - overloadThreshold: { type: 'number', required: false, default: 30 } -- weekly hours above which teacher is "overloaded"
  - underloadThreshold: { type: 'number', required: false, default: 8 } -- weekly hours below which teacher is "underutilized"
  - department: { type: 'string', required: false }
- exports: ['excel', 'pdf']

**Columns:**
- teacherName (string, sortable) -- 'שם מורה'
- classification (string, sortable) -- 'סיווג'
- instruments (string) -- 'כלי נגינה'
- totalWeeklyHours (number, sortable) -- 'סה"כ שעות'
- studentCount (number, sortable) -- 'מספר תלמידים'
- orchestraCount (number, sortable) -- 'מספר תזמורות'
- status (string, sortable) -- 'סטטוס' (values: 'עומס יתר', 'תקין', 'תת ניצול')

**generate(params, scope, { services }) logic:**
1. Build hours_summary filter identical to teacher-hours-summary (tenantId, schoolYearId, scope-based, department-based).
2. Query hours_summary collection.
3. Map each doc to a row:
   - teacherName = `${doc.teacherInfo.lastName} ${doc.teacherInfo.firstName}`
   - classification = doc.teacherInfo.classification
   - instruments = doc.teacherInfo.instruments.join(', ')
   - totalWeeklyHours = doc.totals.totalWeeklyHours
   - studentCount = (doc.breakdown?.students || []).length
   - orchestraCount = (doc.breakdown?.orchestras || []).length
   - status: if totalWeeklyHours > overloadThreshold => 'עומס יתר', if < underloadThreshold => 'תת ניצול', else 'תקין'
4. Summary items:
   - total teachers (number)
   - overloaded count (number) -- label 'עומס יתר'
   - underutilized count (number) -- label 'תת ניצול'
   - average hours (number) -- label 'ממוצע שעות'
5. Return `{ columns: this.columns, rows, summary: { items } }`.
  </action>
  <verify>
Call `GET /api/reports/registry` -- should include 'teacher-workload'. Call `GET /api/reports/teacher-workload?overloadThreshold=25` with auth -- should return rows with status field reflecting the threshold.
  </verify>
  <done>Teacher Workload Distribution generator returns per-teacher workload comparison with overloaded/underutilized status flags and summary counts.</done>
</task>
