  <name>Task 1: Create Teacher Salary Projection generator (TCHR-03)</name>
  <files>api/reports/generators/teacher-salary-projection.generator.js</files>
  <action>
Create `teacher-salary-projection.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'teacher-salary-projection'
- name: 'תחזית שכר מורים' (Teacher Salary Projection)
- description: 'הערכת שכר לפי שעות, סיווג ודרגה'
- category: 'teacher'
- icon: 'CurrencyCircleDollar'
- roles: ['מנהל'] -- salary data restricted to full admins only
- params: { department: { type: 'string', required: false } }
- exports: ['excel', 'pdf']

**Hourly rate lookup table (hardcoded, standard Israeli Ministry of Education rates for music conservatories):**
Define a `HOURLY_RATES` object at the top of the file mapping classification x degree to hourly rate in ILS:
```js
const HOURLY_RATES = {
  'ממשיך': {
    'תואר שלישי': 95,
    'תואר שני': 85,
    'תואר ראשון': 75,
    'מוסמך בכיר': 70,
    'מוסמך': 65,
    'בלתי מוסמך': 55,
    '_default': 65,
  },
  'חדש': {
    'תואר שלישי': 85,
    'תואר שני': 75,
    'תואר ראשון': 65,
    'מוסמך בכיר': 60,
    'מוסמך': 55,
    'בלתי מוסמך': 45,
    '_default': 55,
  },
  '_default': 60,
};
```
Add a helper: `function getHourlyRate(classification, degree)` that looks up the rate from HOURLY_RATES, falling back through classification._default then _default.

**Columns:**
- teacherName (string, sortable) -- 'שם מורה'
- idNumber (string) -- 'ת.ז.'
- classification (string, sortable) -- 'סיווג'
- degree (string, sortable) -- 'דרגה'
- totalWeeklyHours (number, sortable) -- 'שעות שבועיות'
- hourlyRate (currency, sortable) -- 'תעריף לשעה'
- weeklyProjection (currency, sortable) -- 'עלות שבועית'
- monthlyProjection (currency, sortable) -- 'עלות חודשית' (weekly * 4.33)
- annualProjection (currency, sortable) -- 'עלות שנתית' (monthly * 10, school year ~10 months)

**generate(params, scope, { services }) logic:**
1. Query hours_summary collection with scope/schoolYearId/department filters (same pattern as teacher-hours-summary generator).
2. For each doc, also look up the teacher's degree from the teacher collection: query teacher by `_id: ObjectId(doc.teacherId)` and `tenantId: scope.tenantId`, read `professionalInfo.degree`.
3. For efficiency, batch-load all teacher docs for the relevant teacherIds in a single query with projection `{ professionalInfo: 1 }`.
4. Map each hours_summary doc to a row:
   - teacherName = `${doc.teacherInfo.lastName} ${doc.teacherInfo.firstName}`
   - idNumber = doc.teacherInfo.idNumber
   - classification = doc.teacherInfo.classification
   - degree = teacher.professionalInfo?.degree || 'לא צוין'
   - totalWeeklyHours = doc.totals.totalWeeklyHours
   - hourlyRate = getHourlyRate(classification, degree)
   - weeklyProjection = totalWeeklyHours * hourlyRate (round to 2 decimal places)
   - monthlyProjection = weeklyProjection * 4.33 (round to 2 decimal places)
   - annualProjection = monthlyProjection * 10 (round to 2 decimal places)
5. Summary items:
   - total teachers (number)
   - total monthly cost (currency) -- sum of all monthlyProjection
   - total annual cost (currency) -- sum of all annualProjection
   - average hourly rate (currency) -- average of hourlyRate values
6. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Import ObjectId from 'mongodb' for teacher lookup. The hourly rates are approximate reference values -- add a comment noting these are configurable reference rates.
  </action>
  <verify>
Call `GET /api/reports/teacher-salary-projection` with admin auth -- should return rows with hourlyRate, weeklyProjection, monthlyProjection, annualProjection columns all populated with currency values. Summary should show total monthly and annual cost.
  </verify>
  <done>Teacher Salary Projection generator returns per-teacher salary estimates using classification/degree rate lookup, with weekly/monthly/annual projections and cost totals in summary.</done>

<task type="auto">
  <name>Task 2: Create Teacher Roster generator (TCHR-04)</name>
  <files>api/reports/generators/teacher-roster.generator.js</files>
  <action>
Create `teacher-roster.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'teacher-roster'
- name: 'רשימת מורים' (Teacher Roster)
- description: 'רשימת כל המורים עם פרטי קשר, כלי נגינה, תפקידים וסטטוס'
- category: 'teacher'
- icon: 'Users'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - status: { type: 'string', required: false, default: 'all', allowed: ['all', 'active', 'inactive'] }
  - department: { type: 'string', required: false }
- exports: ['excel', 'pdf']

**Columns:**
- teacherName (string, sortable) -- 'שם מורה'
- idNumber (string) -- 'ת.ז.'
- email (string) -- 'אימייל'
- phone (string) -- 'טלפון'
- classification (string, sortable) -- 'סיווג'
- degree (string, sortable) -- 'דרגה'
- instruments (string) -- 'כלי נגינה'
- roles (string) -- 'תפקידים'
- seniority (number, sortable) -- 'ותק'
- isActive (string, sortable) -- 'סטטוס' (display 'פעיל' / 'לא פעיל')

**generate(params, scope, { services }) logic:**
1. Build filter on teacher collection: `{ tenantId: scope.tenantId }`.
2. If params.status === 'active': add `{ isActive: true }`. If 'inactive': `{ isActive: false }`. If 'all': no isActive filter.
3. If params.department: filter teachers whose `professionalInfo.instruments` includes an instrument belonging to that department (use getInstrumentDepartment from constants to build the instrument list for the department, then use `{ 'professionalInfo.instruments': { $in: instrumentsInDepartment } }`).
4. For scope.type === 'department': same instrument-based filtering using scope.departmentIds (build instruments list for all scope departments).
5. For scope.type === 'own': add `{ _id: ObjectId(scope.teacherId) }`.
6. Query teacher collection with the built filter. Sort by lastName ascending.
7. Map each teacher doc to a row:
   - teacherName = `${teacher.personalInfo?.lastName || ''} ${teacher.personalInfo?.firstName || ''}`.trim()
   - idNumber = teacher.personalInfo?.idNumber || ''
   - email = teacher.personalInfo?.email || teacher.credentials?.email || ''
   - phone = teacher.personalInfo?.phone || ''
   - classification = teacher.professionalInfo?.classification || ''
   - degree = teacher.professionalInfo?.degree || ''
   - instruments = (teacher.professionalInfo?.instruments || []).join(', ')
   - roles = (teacher.roles || []).join(', ')
   - seniority = teacher.professionalInfo?.seniority || 0
   - isActive = teacher.isActive ? 'פעיל' : 'לא פעיל'
8. Summary items:
   - total teachers (number)
   - active count (number) -- label 'פעילים'
   - inactive count (number) -- label 'לא פעילים'
   - instruments count (number) -- label 'כלי נגינה שונים' (count distinct instruments across all teachers)
9. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Import `getInstrumentDepartment` from `../../config/constants.js` and `ObjectId` from `mongodb`. Import `INSTRUMENT_MAP` from constants if needed to build instruments-in-department list (filter INSTRUMENT_MAP entries where department matches, map to name).
  </action>
  <verify>
Call `GET /api/reports/teacher-roster` with auth -- should return all teachers. Call with `?status=active` -- should return only active teachers. Call with `?department=כלי קשת` -- should return only string instrument teachers.
  </verify>
  <done>Teacher Roster generator returns complete teacher listing with contact info, qualifications, instruments, roles, and active/inactive filtering.</done>
</task>
