<objective>
Add tenant scoping to the primary cascade deletion service (transaction-based).

Purpose: services/cascadeDeletion.service.js performs cascade deletion with MongoDB transactions. Every query in this file (findOne, find, updateMany, updateOne, insertOne) has ZERO tenantId references. A cascade deletion initiated by Tenant A could modify Teacher/Orchestra/Rehearsal/Theory/Bagrut/Attendance documents belonging to Tenant B. This is a CRITICAL multi-tenant data corruption vulnerability.

Output: Fully tenant-scoped cascadeDeletion.service.js where every DB query includes tenantId.
</objective>
