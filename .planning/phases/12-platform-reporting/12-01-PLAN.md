<task type="auto">
  <name>Task 1: Add reporting service functions to super-admin.service.js</name>
  <files>api/super-admin/super-admin.service.js</files>
  <action>
Add four new exported functions to `superAdminService` in super-admin.service.js. Place them after the existing `getTenantAuditLog` function, under a new section comment `// --- Platform Reporting Methods ---`.

**1. `getReportingTenantList()`** (covers REPT-01 + REPT-02 + REPT-03):

Fetch all tenants from the tenant collection sorted by name. Extract tenantIds using the existing `t.tenantId || t._id.toString()` pattern. Then run six aggregation queries in parallel using `Promise.all()`:

- Teacher counts: aggregate on teacher collection with `{ tenantId: { $in: tenantIds }, isActive: true }`, group by `$tenantId`, sum 1.
- Student counts: same pattern on student collection.
- Orchestra counts: same pattern on orchestra collection, filter `isActive: true`.
- Last admin login per tenant: aggregate on teacher collection matching `{ tenantId: { $in: tenantIds }, isActive: true, roles: 'מנהל' }`, sort by `credentials.lastLogin: -1`, group by `$tenantId` with `$first: '$credentials.lastLogin'` as `lastAdminLogin`.
- Ministry latest snapshots: aggregate on ministry_report_snapshots collection matching `{ tenantId: { $in: tenantIds } }`, sort by `generatedAt: -1`, group by `$tenantId` with `$first: '$generatedAt'` as `latestDate`, `$first: '$completionPercentage'` as `completionPercentage`, `$sum: 1` as `count`.

Build Maps from each result array (key = `_id`, value = count/object). Then map over tenants to enrich each with:

```javascript
{
  ...tenant,
  stats: {
    teacherCount,
    studentCount,
    orchestraCount,
    lastAdminLogin: adminLoginMap.get(tid)?.lastAdminLogin || null,
    teacherUtilization: computeUtilization(teacherCount, tenant.subscription?.maxTeachers),
    studentUtilization: computeUtilization(studentCount, tenant.subscription?.maxStudents),
  },
  ministryStatus: {
    latestSnapshotDate: snapshotMap.get(tid)?.latestDate || null,
    completionPercentage: snapshotMap.get(tid)?.completionPercentage ?? null,
    snapshotCount: snapshotMap.get(tid)?.count || 0,
  },
  alerts: deriveHealthAlerts(tenant, { teacherCount, studentCount }),
}
```

Add a private helper `computeUtilization(count, max)` that returns `null` if `max` is falsy/0, otherwise `Math.round((count / max) * 100)`.

Add a private helper `deriveHealthAlerts(tenant, stats)` that returns an array of alert objects. Logic:
- If `tenant.subscription?.endDate` is truthy and `new Date(tenant.subscription.endDate) <= new Date(Date.now() + 30 * 24 * 60 * 60 * 1000)`: push `{ type: 'expiring_soon', daysRemaining: Math.ceil((endDate - now) / 86400000), severity: daysRemaining <= 7 ? 'critical' : 'warning' }`.
- If `maxTeachers > 0 && stats.teacherCount >= maxTeachers`: push `{ type: 'over_limit_teachers', current: stats.teacherCount, max: maxTeachers, severity: 'warning' }`.
- If `maxStudents > 0 && stats.studentCount >= maxStudents`: push `{ type: 'over_limit_students', current: stats.studentCount, max: maxStudents, severity: 'warning' }`.
- If `!tenant.isActive || !tenant.subscription?.isActive`: push `{ type: 'inactive', severity: 'info' }`.

**2. `getReportingTenantDetail(tenantId)`** (single tenant variant):

Same enrichment as above but for a single tenant. Fetch tenant by `ObjectId.createFromHexString(tenantId)`, throw if not found. Use the string `tid = tenant.tenantId || tenant._id.toString()`. Run count queries (not aggregation) for teacher/student/orchestra with `countDocuments({ tenantId: tid, isActive: true })`. Run the admin login aggregation and snapshot aggregation scoped to just this `tid`. Return the same shape as one item from `getReportingTenantList()`.

**3. `getReportingMinistryStatus()`** (covers REPT-02 standalone):

Fetch all tenants (just `_id`, `name`, `slug`, `tenantId` via projection). Extract tenantIds. Run the same ministry_report_snapshots aggregation as in getReportingTenantList(). Map tenants to:
```javascript
{ tenantId: tid, tenantName: t.name, slug: t.slug, latestSnapshotDate, completionPercentage, snapshotCount }
```
Default to `null`/`0` for tenants with no snapshots.

**4. `getReportingDashboard()`** (covers REPT-04):

Run `Promise.all([getReportingOverviewCards(), getReportingTenantList()])`.

`getReportingOverviewCards()` is a new private function that extends the existing `getPlatformAnalytics()` pattern but adds `totalOrchestras` (count orchestras with `isActive: true`) and `alertCount` (computed after tenantHealth is available -- see below). Since we need alertCount from the tenant list, the dashboard function should:
1. Call `getReportingTenantList()` to get full tenant health.
2. Compute overview from the tenant list: totalTenants, activeTenants (count where isActive), totalTeachers/Students/Orchestras (sum from stats), subscriptionsByPlan (aggregate from tenant.subscription.plan).
3. Flatten alerts: `tenantHealth.filter(t => t.alerts.length > 0).flatMap(t => t.alerts.map(a => ({ ...a, tenantId: t._id?.toString() || t.tenantId, tenantName: t.name })))`.

Return `{ overview, tenantHealth, alerts }`.

**Important:** Use `getCollection(COLLECTIONS.ORCHESTRA)` and `getCollection(COLLECTIONS.MINISTRY_REPORT_SNAPSHOTS)` -- import COLLECTIONS from constants.js (already imported). Do NOT duplicate the existing `getTenantsWithStats()` or `getPlatformAnalytics()` -- the new functions are additive. Add all four functions to the `superAdminService` export object.
