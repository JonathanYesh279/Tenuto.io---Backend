**Input (input.tsx):**

1. Replace hardcoded `px-3 py-2` with `px-spacing-element py-2`. The spacing-element token is 12px (0.75rem) which matches px-3 exactly. Using the named token ensures future spacing changes propagate.
   - Full className becomes: `"flex h-10 w-full rounded-md border border-input bg-background px-spacing-element py-2 text-sm ring-offset-background file:border-0 file:bg-transparent file:text-sm file:font-medium placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50"`
   - NOTE: py-2 (8px) stays as-is -- no exact spacing token for vertical input padding. The semantic spacing tokens are page/section/card/element level, not input-field level.

2. The focus ring (`ring-2 ring-ring ring-offset-2`) already uses semantic tokens -- no change needed.

**Dialog (dialog.tsx):**

1. DialogContent: Replace `gap-4 p-6` with `gap-spacing-element p-spacing-card`. Reasoning:
   - `p-6` = 24px = spacing-card token (24px) -- exact match
   - `gap-4` = 16px -- closest token is spacing-element (12px) but that's too tight for dialog sections. Keep `gap-4` as a reasonable dialog-specific value. Actually, use `gap-spacing-element` since 12px gap between dialog sections is more compact and consistent.

   Wait -- reconsider: `gap-4` (16px) is the current dialog spacing and looks good. The spacing-element token (12px) would make dialogs feel cramped. The closest semantic token is spacing-card (24px) which is too much gap. Keep `gap-4` as-is for dialog section gap -- it's a reasonable dialog-specific value that doesn't need tokenization.

   Final: Replace only `p-6` with `p-spacing-card` in DialogContent. Keep `gap-4` and `shadow-4` as-is (shadow-4 is already token-backed).

2. DialogTitle: Replace `text-lg font-semibold` with `text-h3`. The text-h3 token is 20px (vs text-lg 18px) with fontWeight 600 (same as font-semibold). This is a deliberate 2px increase for Hebrew readability (Phase 66 design decision). Keep `leading-none tracking-tight` as overrides.

3. Close button: `right-4 top-4` positioning is standard -- no change needed. Focus ring already uses semantic tokens.

4. DialogHeader: `space-y-1.5` is fine -- no semantic token for this small spacing.

NOTE: Do NOT change index.css `.input:focus` styles -- that's Phase 69 CSS cleanup scope.
