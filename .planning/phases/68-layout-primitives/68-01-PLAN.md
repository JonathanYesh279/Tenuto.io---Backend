  <action>
Create `src/components/layout/` directory.

**PageShell.tsx** — Page-level wrapper providing max-width and vertical section spacing. Follow the CVA + cn() + forwardRef pattern from Card.tsx.

CVA variants for `width`:
- `constrained`: `"max-w-7xl mx-auto"` (default) — standard content width
- `narrow`: `"max-w-4xl mx-auto"` — forms and detail pages
- `full`: `""` — no max-width (dashboards, schedules)

Base classes: `"space-y-spacing-section"` (32px gap between direct children).

CRITICAL: Do NOT add any padding. Layout.tsx already provides `p-6` (24px) on the content wrapper. Adding padding here causes a double-padding bug (observed in Profile.tsx).

Use `<div>` element (not `<main>` — Layout.tsx already has `<main>`).

Interface extends `React.HTMLAttributes<HTMLDivElement>` and `VariantProps<typeof pageShellVariants>`. Only `children: React.ReactNode` as the required prop.

Export both `PageShell` and `pageShellVariants`.

**SectionWrapper.tsx** — Section-level wrapper with optional heading and element spacing. No CVA needed (no variants).

Props: `title?: string`, `description?: string`, `children: React.ReactNode`. Interface extends `React.HTMLAttributes<HTMLElement>`.

Use `<section>` element for semantic HTML. Apply `space-y-spacing-element` (12px) to the section for children spacing.

When `title` or `description` provided, render a header div BEFORE children:
- `title` renders as `<h2 className="text-h2">` (uses Phase 66 type scale token)
- `description` renders as `<p className="text-small text-muted-foreground">`

Use cn() + forwardRef pattern. Export `SectionWrapper`.
  </action>

Run `npx tsc --noEmit --pretty` from the frontend directory. Both files should compile with zero type errors. Verify files exist: `ls src/components/layout/PageShell.tsx src/components/layout/SectionWrapper.tsx`.
  </verify>
  <done>
PageShell renders a div with max-width variants and space-y-spacing-section, no padding. SectionWrapper renders a section with optional h2 title and space-y-spacing-element. Both accept className and children with no required data props.
  </done>

**FormLayout.tsx** — Form field grid wrapper. Follow CVA + cn() + forwardRef pattern.

CVA variants for `columns`:
- `1`: `"grid-cols-1"` — single column (narrow forms)
- `2`: `"grid-cols-1 md:grid-cols-2"` (default) — responsive 2-column
- `3`: `"grid-cols-1 md:grid-cols-2 lg:grid-cols-3"` — responsive 3-column

Base classes: `"grid gap-spacing-element"` (12px gap between grid cells).

Use `<div>` element. Interface extends `React.HTMLAttributes<HTMLDivElement>` and `VariantProps<typeof formLayoutVariants>`. Only `children` is required.

IMPORTANT: FormLayout handles field grid layout ONLY. No submit buttons, error banners, collapsible sections, or form state. Keep it under 30 lines of actual component code.

Export both `FormLayout` and `formLayoutVariants`.
