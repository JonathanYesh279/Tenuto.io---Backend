<task type="auto">
  <name>Task 1: Excel shaper module</name>
  <files>api/reports/report.excel-shaper.js</files>
  <action>
Create `api/reports/report.excel-shaper.js` that exports a `shapeExcel(reportMeta, generatorOutput)` function.

Import ExcelJS (already in package.json). The function receives:
- `reportMeta` — `{ id, name }` from the generator
- `generatorOutput` — raw generator return: `{ columns, rows, summary, columnGroups? }`

Implementation:
1. Create a new ExcelJS Workbook with `creator: 'Tenuto.io'`, `created: new Date()`
2. Add a single worksheet named after `reportMeta.name` (truncate to 31 chars for Excel limit)
3. Set worksheet `rightToLeft = true` and `views: [{ rightToLeft: true }]`
4. **Header row:** Write column labels (from `generatorOutput.columns[].label`) as the first row. Style: bold, fill color `#2563EB` (blue), white font color, font size 11, border bottom.
5. **Data rows:** Iterate `generatorOutput.rows`. For each row, write cell values keyed by `columns[].key`. Apply formatting per column type:
   - `string`: no special format
   - `number`: `numFmt: '#,##0'` (thousands separator)
   - `percentage`: `numFmt: '0.0%'` — divide value by 100 if > 1 (generators store as 0-100 not 0-1)
   - `currency`: `numFmt: '#,##0 ₪'`
   - `date`: `numFmt: 'DD/MM/YYYY'`
6. **Summary row:** After all data rows, add a blank row, then a summary row. For each `summary.items[]`, find the best matching column by label similarity or place in sequential cells. Style: bold, fill color `#F3F4F6` (light gray), top border.
7. **Auto-width columns:** After populating, iterate columns and set width = max(header label length, longest data cell value length) + 2, capped at 40.
8. Return `workbook.xlsx.writeBuffer()` (returns a Promise of Buffer).

Do NOT import or depend on report.contract.js shapeResponse — this is intentionally separate shaping logic (EXPO-03).
  </action>
  <verify>Create a small inline test at the bottom of the file (commented out) showing expected input/output shape. Verify file has no syntax errors: `node -e "import('./api/reports/report.excel-shaper.js').then(m => console.log(Object.keys(m)))"`</verify>
  <done>shapeExcel function exists, accepts (reportMeta, generatorOutput), returns a Promise of Buffer containing valid .xlsx data</done>
</task>

<task type="auto">
  <name>Task 2: Export controller with Excel handler</name>
  <files>api/reports/report.export.controller.js</files>
  <action>
Create `api/reports/report.export.controller.js` that exports `reportExportController` with an `exportExcel` method.

The controller:
1. Import `reportOrchestrator` from `./report.orchestrator.js`
2. Import `shapeExcel` from `./report.excel-shaper.js`
3. Import `getGenerator` from `./report.registry.js`

`exportExcel(req, res, next)`:
1. Extract `reportId` from `req.params`
2. Call `reportOrchestrator.generateReport(reportId, { ...req.query, limit: '99999', page: '1' }, req.context)` — this returns ALL rows (effectively unpaginated) through the existing pipeline including scope/validation
3. Get generator via `getGenerator(reportId)` for metadata (name, id)
4. Check `generator.exports.includes('excel')` — if not, return 400 with `{ error: 'Excel export not supported for this report', code: 'EXPORT_NOT_SUPPORTED' }`
5. Build `generatorOutput` from the orchestrator result: `{ columns: result.columns, rows: result.rows, summary: result.summary, columnGroups: result.columnGroups }`
6. Call `const buffer = await shapeExcel({ id: generator.id, name: generator.name }, generatorOutput)`
7. Set response headers:
   - `Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
   - `Content-Disposition: attachment; filename="${encodeURIComponent(generator.name)}.xlsx"`
   - `Content-Length: buffer.length`
8. `res.send(buffer)`
9. Wrap in try/catch, call `next(err)` on failure

Also stub out `exportPdf` as a placeholder that returns 501 `{ error: 'PDF export coming soon', code: 'NOT_IMPLEMENTED' }` — Plan 54-02 will implement it.
  </action>
  <verify>`node -e "import('./api/reports/report.export.controller.js').then(m => console.log(Object.keys(m.reportExportController)))"`  should print `['exportExcel', 'exportPdf']`</verify>
  <done>Export controller exists with exportExcel that calls orchestrator (unpaginated) then shapeExcel, and returns .xlsx buffer with correct headers</done>
