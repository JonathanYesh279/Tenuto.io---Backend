<task type="auto">
  <name>Task 1: Install pdfkit and create PDF shaper module</name>
  <files>api/reports/report.pdf-shaper.js</files>
  <action>
First install pdfkit: `npm install pdfkit`

Then create `api/reports/report.pdf-shaper.js` that exports a `shapePdf(reportMeta, generatorOutput, options)` function.

The function receives:
- `reportMeta` — `{ id, name }` from the generator
- `generatorOutput` — `{ columns, rows, summary }`
- `options` — `{ conservatoryName, generatedAt }` (conservatory name from tenant doc)

Implementation using PDFKit:
1. Create a new PDFDocument with `size: 'A4', layout: 'landscape'` (landscape better for wide tables), `margins: { top: 60, bottom: 60, left: 40, right: 40 }`, `bufferPages: true`
2. Collect output into a Buffer using stream pattern: `const chunks = []; doc.on('data', c => chunks.push(c)); doc.on('end', () => resolve(Buffer.concat(chunks)));`
3. **Page header function** `drawHeader(doc)`: At y=20, draw conservatory name (right-aligned for RTL, font size 14, bold), report title (reportMeta.name, font size 12), and generation date (font size 9, gray). Draw a horizontal line at y=55.
4. **Page footer function** `drawFooter(doc, pageNum, totalPages)`: At bottom margin, draw a horizontal line, then centered text `עמוד ${pageNum} מתוך ${totalPages}` (font size 8, gray).
5. **Table rendering:**
   - Calculate column widths: distribute available page width (A4 landscape ~ 760 usable) proportionally. Give string columns 1.5x weight, number/percentage/currency 1x weight. Min width 50, max width 200.
   - **Table header row:** Fill background `#2563EB`, white text, font size 9 bold. Draw each cell right-aligned (Hebrew RTL). Row height 25.
   - **Data rows:** Alternate row fill (`#FFFFFF` and `#F9FAFB`). Font size 8. Format values per type:
     - `number`: toLocaleString('he-IL')
     - `percentage`: `${value}%`
     - `currency`: `₪${value.toLocaleString('he-IL')}`
     - `date`: format as DD/MM/YYYY
     - `string`: as-is
   - Row height 20. Before each row, check if y + rowHeight > page height - bottom margin. If so, call `doc.addPage()` and redraw header + table header row.
6. **Summary section:** After all data rows, add 15px gap, draw a gray background row with summary items. Each item: `${item.label}: ${item.value}` in bold.
7. **Finalize:** After all content, use `doc.bufferedPageRange()` to get total pages. Loop through pages with `doc.switchToPage(i)` and call `drawFooter(doc, i+1, totalPages)` on each.
8. Call `doc.end()` and return the Promise of Buffer.

For Hebrew font: PDFKit's built-in Helvetica does not support Hebrew. Register a font that supports Hebrew. Check if `public/fonts/` has any Hebrew-capable font file (TTF/OTF). If available, use `doc.registerFont('Hebrew', 'path/to/font.ttf')`. If not, use the Reisinger-Yonatan font at `public/fonts/Reisinger-Yonatan-web/Reisinger-Yonatan-Regular.ttf` (exists per git status showing deleted MACOSX files for this font). If that font file does not exist or is not embeddable, fall back to embedding a system font or noting that Hebrew rendering may be limited — add a code comment about this.

<task type="auto">
  <name>Task 2: Wire PDF export into controller</name>
  <files>api/reports/report.export.controller.js</files>
  <action>
Update `api/reports/report.export.controller.js` to replace the 501 stub `exportPdf` with a working implementation.

1. Import `shapePdf` from `./report.pdf-shaper.js`
2. Import `getCollection` from `../../services/mongoDB.service.js`
3. Import `ObjectId` from `mongodb` for tenant lookup

`exportPdf(req, res, next)`:
1. Extract `reportId` from `req.params`
2. Call `reportOrchestrator.generateReport(reportId, { ...req.query, limit: '99999', page: '1' }, req.context)` — same pattern as exportExcel
3. Get generator via `getGenerator(reportId)`, check `generator.exports.includes('pdf')` — if not, return 400
4. Fetch conservatory name: `const tenant = await getCollection('tenant').findOne({ _id: new ObjectId(req.context.tenantId) }, { projection: { name: 1 } })`. Use `tenant?.name || 'קונסרבטוריון'` as fallback.
5. Build `generatorOutput` from orchestrator result (same as exportExcel)
6. Call `const buffer = await shapePdf({ id: generator.id, name: generator.name }, generatorOutput, { conservatoryName: tenant?.name || 'קונסרבטוריון', generatedAt: new Date() })`
7. Set response headers:
   - `Content-Type: application/pdf`
   - `Content-Disposition: attachment; filename="${encodeURIComponent(generator.name)}.pdf"`
   - `Content-Length: buffer.length`
8. `res.send(buffer)`

The route is already wired from Plan 54-01 (the `/export/pdf` route points to `reportExportController.exportPdf`).

  <verify>`node -e "import('./api/reports/report.export.controller.js').then(m => console.log(Object.keys(m.reportExportController)))"` should print `['exportExcel', 'exportPdf']`. Verify no 501 stub remains: `grep -c 'NOT_IMPLEMENTED' api/reports/report.export.controller.js` should return 0.</verify>
