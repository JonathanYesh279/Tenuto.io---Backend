  <name>Task 1: Create theory-course.service.js with CRUD, roster management, and analytics</name>
  <files>api/theory/theory-course.service.js</files>
  <action>
Create a new service file `api/theory/theory-course.service.js` following the pattern of `orchestra.service.js` (parent-child entity management). Import `getCollection` from `../../services/mongoDB.service.js`, `ObjectId` from `mongodb`, and date helpers from `../../utils/dateHelpers.js`.

Export a `theoryCourseService` object with these functions:

1. **createCourse(courseData, options)** — Validate with `validateTheoryCourse` (from theory.validation.js). Insert into `theory_course` collection with fields: tenantId, category, teacherId, dayOfWeek, startTime, endTime, location, studentIds[], lessonIds[] (initially empty), schoolYearId, startDate, endDate, excludeDates[], notes, syllabus, isActive (default true), createdAt, updatedAt. Use `requireTenantId(options.context?.tenantId)` pattern. After insert, update teacher record: `$addToSet: { 'teaching.theoryCourseIds': courseId }`. Return the created course document.

2. **getCourses(filterBy, options)** — Query `theory_course` collection with tenantId scoping. Support filters: category, teacherId, schoolYearId, isActive. Sort by category ascending. Return array of course documents.

3. **getCourseById(courseId, options)** — Fetch single course by _id and tenantId. Throw if not found.

4. **updateCourse(courseId, updateData, options)** — findOneAndUpdate with tenantId scoping. Only allow updating: category, teacherId, dayOfWeek, startTime, endTime, location, notes, syllabus, isActive. Always set updatedAt. Return updated document.

5. **deleteCourse(courseId, options)** — Delete the course document. Set `courseId: null` on all linked theory_lesson documents (`{ courseId: courseId }` -> `$set: { courseId: null }`). Remove courseId from teacher's `teaching.theoryCourseIds` via `$pull`. Remove courseId from students' `enrollments.theoryCourseIds` via `$pull`. Return deleted count.

6. **addStudentToCourse(courseId, studentId, options)** — `$addToSet` studentId to course's `studentIds[]`. Also `$addToSet` studentId to future lessons' `studentIds[]` (where `courseId` matches AND `date >= now()`). Update student: `$addToSet: { 'enrollments.theoryCourseIds': courseId }`. Return updated course.

7. **removeStudentFromCourse(courseId, studentId, options)** — `$pull` studentId from course's `studentIds[]`. Also `$pull` from future lessons' `studentIds[]`. Update student: `$pull: { 'enrollments.theoryCourseIds': courseId }`. Return updated course.

8. **getCourseAttendanceAnalytics(courseId, options)** — Fetch course to get lessonIds[]. Run two aggregation pipelines on `activity_attendance`:
   - Per-student stats: group by `$studentId` where `sessionId: { $in: course.lessonIds }` and `activityType: 'תאוריה'`. Compute totalSessions, attended (status in ['הגיע/ה', 'איחור']), absences (status 'לא הגיע/ה'), lateCount (status 'איחור'), attendanceRate percentage.
   - Per-session stats: group by `$sessionId`, compute date, totalStudents, presentCount.
   Return { courseId, category, totalLessons, totalStudents, studentStats, sessionStats }.

9. **linkLessonsToCourse(courseId, lessonIds, options)** — `$addToSet: { lessonIds: { $each: lessonIds } }` on course. `$set: { courseId }` on each lesson document. Used by enhanced bulkCreate.

Use `requireTenantId` from existing pattern (check theory.service.js imports). Use `toUTC(now())` for timestamps. Follow error handling pattern from theory.service.js (try/catch with descriptive error messages).

IMPORTANT: For attendance status matching, use 'איחור' (not 'איחר/ה') — verify against existing activity_attendance data by checking what `updateTheoryAttendance` in theory.service.js actually stores. Check both 'איחור' and 'איחר/ה' patterns in the codebase and use whichever is canonical.

**Part A — Validation (theory.validation.js):**

Add a new `theoryCourseSchema` Joi schema after the existing `theoryBulkCreateSchema`. Fields:
- `tenantId`: Joi.any().strip() (same pattern as theoryLessonSchema)
- `category`: Joi.string().valid(...VALID_THEORY_CATEGORIES).required()
- `teacherId`: Joi.string().required()
- `dayOfWeek`: Joi.number().integer().min(0).max(6).required()
- `startTime`: Joi.string().pattern(/^([0-1]?[0-9]|2[0-3]):[0-5][0-9]$/).required()
- `endTime`: Joi.string().pattern(/^([0-1]?[0-9]|2[0-3]):[0-5][0-9]$/).required()
- `location`: Joi.string().required()
- `studentIds`: Joi.array().items(Joi.string().hex().length(24)).default([])
- `schoolYearId`: Joi.string().required()
- `startDate`: Joi.date().required()
- `endDate`: Joi.date().min(Joi.ref('startDate')).required()
- `excludeDates`: Joi.array().items(Joi.date()).default([])
- `notes`: Joi.string().allow('', null).default('')
- `syllabus`: Joi.string().allow('', null).default('')
- `isActive`: Joi.boolean().default(true)
- `createdAt`, `updatedAt`: Joi.date().default(() => new Date())

Include the same start/end time custom validation (endTime must be after startTime).

Add and export `validateTheoryCourse(courseData)` function using same pattern as `validateTheoryLesson`.

Also add a `theoryCourseUpdateSchema` (partial — all fields optional except tenantId strip) and export `validateTheoryCourseUpdate`.

**Part B — Migration script (scripts/migrate-theory-courses.js):**

Create a standalone migration script that:
1. Connects to MongoDB using the project's connection pattern (import `connectToMongo` from `../services/mongoDB.service.js` and `getCollection`)
2. Adds `courseId: null` to all `theory_lesson` documents that don't have a `courseId` field: `updateMany({ courseId: { $exists: false } }, { $set: { courseId: null } })`
3. Creates indexes on `theory_course` collection:
   - `{ tenantId: 1, category: 1, schoolYearId: 1 }`
   - `{ tenantId: 1, teacherId: 1 }`
   - `{ tenantId: 1, isActive: 1 }`
4. Creates index on `theory_lesson` collection:
   - `{ tenantId: 1, courseId: 1 }`
5. Verifies index on `activity_attendance`:
   - `{ tenantId: 1, sessionId: 1, activityType: 1 }` (create if not exists)
6. Logs results: how many lessons updated, how many indexes created.
7. Does NOT auto-group existing lessons (per research recommendation — let user decide).

Script should be runnable with `node scripts/migrate-theory-courses.js` and be idempotent (safe to run multiple times).
