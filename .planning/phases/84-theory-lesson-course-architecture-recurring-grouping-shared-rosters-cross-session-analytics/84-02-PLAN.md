**Part A — Controller (theory.controller.js):**

Import `theoryCourseService` from `./theory-course.service.js` at the top.

Add these controller functions to the `theoryController` export object:

1. **getCourses(req, res, next)** — Extract filters from req.query (category, teacherId, schoolYearId, isActive). Call `theoryCourseService.getCourses(filterBy, { context: req.context })`. Return 200 with courses array.

2. **getCourseById(req, res, next)** — Get courseId from `req.params.id`. Call `theoryCourseService.getCourseById(courseId, { context: req.context })`. Return 200 with course. Return 404 if not found.

3. **createCourse(req, res, next)** — Call `theoryCourseService.createCourse(req.body, { context: req.context })`. Return 201 with created course.

4. **updateCourse(req, res, next)** — Get courseId from `req.params.id`. Call `theoryCourseService.updateCourse(courseId, req.body, { context: req.context })`. Return 200 with updated course.

5. **deleteCourse(req, res, next)** — Get courseId from `req.params.id`. Call `theoryCourseService.deleteCourse(courseId, { context: req.context })`. Return 200 with success message.

6. **addStudentToCourse(req, res, next)** — Get courseId from `req.params.id`, studentId from `req.params.studentId`. Call `theoryCourseService.addStudentToCourse(courseId, studentId, { context: req.context })`. Return 200 with updated course.

7. **removeStudentFromCourse(req, res, next)** — Same params. Call `theoryCourseService.removeStudentFromCourse(...)`. Return 200.

8. **getCourseAttendanceAnalytics(req, res, next)** — Get courseId from `req.params.id`. Call `theoryCourseService.getCourseAttendanceAnalytics(courseId, { context: req.context })`. Return 200 with analytics object.

Use `sendSuccessResponse` and `sendErrorResponse` from `../../utils/errorResponses.js` (same pattern as existing controller functions). Wrap each in try/catch.

**Part B — Routes (theory.route.js):**

Add course routes BEFORE the existing `/:id` routes (to avoid route parameter conflicts where "courses" would match `:id`). Add them in a logical group:

```
// Course routes
router.get('/courses', requirePermission('theory', 'view'), theoryController.getCourses);
router.get('/courses/:id', requirePermission('theory', 'view'), theoryController.getCourseById);
router.get('/courses/:id/analytics', requirePermission('theory', 'view'), theoryController.getCourseAttendanceAnalytics);
router.post('/courses', requirePermission('theory', 'create'), validateCourseCreate, theoryController.createCourse);
router.put('/courses/:id', requirePermission('theory', 'update'), validateCourseUpdate, theoryController.updateCourse);
router.delete('/courses/:id', requirePermission('theory', 'delete'), theoryController.deleteCourse);
router.post('/courses/:id/student/:studentId', requirePermission('theory', 'create'), theoryController.addStudentToCourse);
router.delete('/courses/:id/student/:studentId', requirePermission('theory', 'delete'), theoryController.removeStudentFromCourse);
```

**Part A — Enhanced bulkCreate (theory.service.js):**

Modify the `bulkCreateTheoryLessons` function to support an optional `createCourse: true` flag in the bulkData payload. The enhancement is backward compatible:

1. At the start of bulkCreateTheoryLessons, after validation, check if `bulkData.createCourse === true`.

2. If `createCourse` is true:
   a. Import `theoryCourseService` dynamically using `const { theoryCourseService } = await import('./theory-course.service.js')` (dynamic import to avoid circular dependency issues — same pattern used in Phase 73/74).
   b. BEFORE creating lessons, create the course entity by calling:
      ```
      const course = await theoryCourseService.createCourse({
        category, teacherId, dayOfWeek, startTime, endTime, location,
        studentIds, schoolYearId, startDate, endDate, excludeDates,
        notes, syllabus
      }, options);
      ```
   c. Add `courseId: course._id.toString()` to each lesson document in the `theoryLessons` array (the map that creates lesson objects from utcDates).
   d. AFTER lessons are inserted successfully, call:
      ```
      await theoryCourseService.linkLessonsToCourse(course._id.toString(), result.theoryLessonIds, options);
      ```
   e. Add `courseId: course._id.toString()` to the return object.

3. If `createCourse` is false/undefined: behavior is EXACTLY as before. Lessons get no `courseId` field (or `courseId: null` for explicit clarity). No course entity is created. This preserves backward compatibility.

**Part B — Update bulk create validation (theory.validation.js):**

Add `createCourse: Joi.boolean().default(false)` to the `theoryBulkCreateSchema`. This is optional and defaults to false, so existing callers are unaffected.
