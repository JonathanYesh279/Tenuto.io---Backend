  <name>Task 1: Create useStudentDashboardData hook</name>
  <files>src/features/students/details/hooks/useStudentDashboardData.ts</files>
  <action>
Create a custom hook that takes `studentId: string` and the already-fetched `student` object, then fetches additional dashboard data and merges everything into a unified return value.

**Data sources to aggregate:**
1. **Student document** (passed in as prop, already fetched by parent): personalInfo, academicInfo, teacherAssignments, orchestraEnrollments, isActive, registrationDate, startDate
2. **Attendance summary** (fetch): `apiService.attendanceAlerts.getStudentSummary(studentId)` -- returns totalSessions, attendedCount, lateCount, absentCount, attendanceRate, consecutiveAbsences, recentHistory
3. **Attendance stats** (fetch): `apiService.analytics.getStudentAttendanceStats(studentId, { months: 6 })` -- for monthly trend data
4. **Orchestras list** (fetch): `apiService.orchestras.getAll()` or from cache -- needed to resolve orchestra names from orchestraEnrollments IDs
5. **Theory lessons** (fetch): `apiService.theoryLessons.getStudentTheoryLessons(studentId)` -- for enrollment table and theory count

**Weekly hours calculation:**
Derive daily lesson hours from `teacherAssignments`. Each assignment has `dayOfWeek` (or needs time block lookup). Build array: `[{ day: 'ראשון', hours: N }, { day: 'שני', hours: N }, ...]` for Sun-Fri (Hebrew day names). Sum `weeklyHours` from each active assignment grouped by dayOfWeek. If dayOfWeek is missing, skip that assignment (don't break).

**Return type (TypeScript interface):**
```typescript
interface StudentDashboardData {
  // Profile
  student: any
  isLoading: boolean

  // Attendance
  attendanceSummary: { totalSessions: number; attendedCount: number; lateCount: number; absentCount: number; attendanceRate: number } | null
  attendanceLoading: boolean
  monthlyAttendance: Array<{ month: string; rate: number }> // last 6 months

  // Weekly hours
  weeklyHours: Array<{ day: string; hours: number }> // Sun-Fri
  totalWeeklyHours: number

  // Enrollments (unified)
  enrollments: Array<{
    id: string
    type: 'individual' | 'orchestra' | 'theory'
    name: string
    instrument: string
    dayTime: string
    room: string
    status: string
  }>

  // Counts
  orchestraCount: number
  theoryCount: number

  // Teacher map for child components
  teacherMap: Record<string, { firstName: string; lastName: string; instrument: string }>
}
```

**Important:** Use `useEffect` + `useState` pattern (not TanStack Query) to match the existing page pattern. Wrap all fetches in try/catch -- individual failures should not break the whole hook. Set partial data on success, null on failure.

**Centralize teacher fetching:** If `teacherAssignments` has teacher IDs, batch-fetch teacher details once (or use the teacher list endpoint). Store in `teacherMap` so ProfileCard and other components can resolve teacher names without N+1 calls. If teacher details are already embedded in teacherAssignments (check the student doc structure), skip the extra fetch.

**Hebrew day names mapping:**
```typescript
const DAY_NAMES: Record<number, string> = {
  0: 'ראשון', 1: 'שני', 2: 'שלישי', 3: 'רביעי', 4: 'חמישי', 5: 'שישי'
}
```
