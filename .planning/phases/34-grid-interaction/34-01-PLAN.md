**RoomSchedule.tsx changes:**

1. Add imports: `useAuth` from `@/services/authContext.jsx`, `tenantService` from `@/services/apiService`, `FilterBar` and `Filters` type from `./FilterBar`.

2. Add filter state:
```typescript
const [filters, setFilters] = useState<Filters>({
  teacherName: '',
  roomName: '',
  activityTypes: ['timeBlock', 'rehearsal', 'theory'],
})
```

3. Add tenant rooms state + fetch:
```typescript
const { user } = useAuth()
const [tenantRooms, setTenantRooms] = useState<Array<{ name: string; isActive: boolean }>>([])

useEffect(() => {
  if (user?.tenantId) {
    tenantService.getRooms(user.tenantId)
      .then((rooms) => setTenantRooms(rooms || []))
      .catch((err) => console.error('Error fetching rooms:', err))
  }
}, [user?.tenantId])
```

4. Add `filteredRooms` useMemo that:
   - Starts with `schedule?.rooms || []`
   - Filters by room name: if `filters.roomName` is non-empty, keep only rooms where `room.room === filters.roomName`
   - Filters activities within each room: filter by `filters.activityTypes.includes(activity.source)` AND (if `filters.teacherName` is non-empty) `activity.teacherName.includes(filters.teacherName)`
   - Removes rooms with no matching activities UNLESS the room was explicitly selected by room filter
   - Merges empty rooms: for each active tenant room NOT already in schedule rooms, add `{ room: roomName, activities: [], hasConflicts: false }`. Only include empty rooms when no room filter is active OR the room matches the filter.
   - Sort all rooms alphabetically by room name using `.sort((a, b) => a.room.localeCompare(b.room, 'he'))`
   - Dependencies: `[schedule, filters, tenantRooms]`

5. Extract room names list for FilterBar dropdown:
```typescript
const roomNames = useMemo(() => {
  const names = new Set<string>()
  for (const room of schedule?.rooms || []) names.add(room.room)
  for (const tr of tenantRooms) if (tr.isActive) names.add(tr.name)
  return Array.from(names).sort((a, b) => a.localeCompare(b, 'he'))
}, [schedule, tenantRooms])
```

6. Update JSX: Add `<FilterBar>` between the page header and SummaryBar:
```tsx
<FilterBar filters={filters} onFiltersChange={setFilters} rooms={roomNames} />
```

7. Pass `filteredRooms` to `<RoomGrid>` instead of `schedule?.rooms || []`.

8. Update stats computation `useMemo` to use `filteredRooms` for occupied/free slot counting so stats reflect filtered view.

**RoomGrid.tsx changes:**

This is still correct when ALL rooms are empty. But when some rooms have activities and some are empty, the empty rooms should render as rows with all empty cells. This already works because the component maps over `rooms` and renders cells for each -- a room with `activities: []` will render with only background cells and no activity cells, which is correct.
- No changes needed to RoomGrid.tsx logic itself; the empty rooms flow through naturally from the filteredRooms data.

**DaySelector.tsx change:**
- Import `DAY_NAMES` from `./utils` instead of defining it locally. This avoids duplication now that utils.ts has the canonical array.
