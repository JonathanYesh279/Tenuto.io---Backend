1. In `student.service.js`, locate the `updateStudent` function's `teacherAssignmentsSyncRequired` block. After the relationship validation section (around line 419, after the `try/catch` for `validateStudentTeacherRelationships`), add a fire-and-forget hours recalculation hook:

```javascript
    // Auto-recalculate hours for affected teachers (fire-and-forget, non-fatal)
    if (teacherAssignmentsSyncRequired) {
      const affectedTeacherIds = new Set([
        ...(originalStudent.teacherAssignments || []).map(a => a.teacherId?.toString()),
        ...(newAssignments || []).map(a => a.teacherId?.toString()),
      ].filter(Boolean));

      if (affectedTeacherIds.size > 0) {
        // Dynamic import to avoid circular dependency (same pattern as Phase 73 import hook)
        import('../hours-summary/hours-summary.service.js').then(({ hoursSummaryService }) => {
          // Get schoolYearId from context or the student's current school year
          const schoolYearId = context?.schoolYearId || originalStudent.schoolYearId;

          for (const tid of affectedTeacherIds) {
            hoursSummaryService.calculateTeacherHours(tid, schoolYearId, { context })
              .then(() => console.log(`AUTO-RECALC: Hours recalculated for teacher ${tid}`))
              .catch(err => console.error(`AUTO-RECALC: Hours recalc failed for teacher ${tid}:`, err.message));
          }
        }).catch(err => console.error('AUTO-RECALC: Failed to load hoursSummaryService:', err.message));
      }
    }
```

2. IMPORTANT: Use dynamic `import()` (not static `require` or top-level `import`) to avoid circular dependency issues. This is the same pattern used in Phase 73's import hook (`api/import/import.service.js`).

3. IMPORTANT: This block must be OUTSIDE the MongoDB session/transaction — after the main update has committed. The hours recalculation should NOT be part of the transaction. Place it after the transaction commit / after the relationship validation block, NOT inside any `try` that would roll back.

4. IMPORTANT: Do NOT use `await` on the `import()` call — let it run fire-and-forget. The student update response should not be blocked or delayed by hours recalculation.

5. Extract `context` from the function parameters. The `updateStudent` function receives `(studentId, value, userInfo)` where `userInfo` might contain context. Check the function signature and find where `context` is available — it should be in `userInfo.context` or passed directly. Use whatever is available for the `schoolYearId`.
