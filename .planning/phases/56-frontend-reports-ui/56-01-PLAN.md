    Create a generic table renderer that can display ANY report from the backend contract `{ columns, rows, summary, metadata }`.

    **Props interface:**
    ```typescript
    interface DefaultTableRendererProps {
      columns: Array<{ key: string; label: string; type: 'string' | 'number' | 'date' | 'percentage' | 'currency' }>
      columnGroups?: Array<{ label: string; columns: string[] }>
      rows: Record<string, any>[]
      summary: { items: Array<{ label: string; value: any; type: string }> }
      metadata: {
        reportId: string
        title: string
        pagination: { page: number; limit: number; totalCount: number; totalPages: number }
        filters: Record<string, any>
        exports: string[]
      }
      sortBy?: string
      sortOrder?: 'asc' | 'desc'
      onSortChange: (sortBy: string, sortOrder: 'asc' | 'desc') => void
      onPageChange: (page: number) => void
      onLimitChange: (limit: number) => void
    }
    ```

    **Table rendering:**
    - Use an HTML table (not the existing Table component — it's too opinionated with actions/view/delete). Build a clean table with Tailwind classes matching the existing design system (border-border, bg-card, text-foreground, etc).
    - Column headers: render each column's `label`. If `columnGroups` is provided, render a grouped header row above with colspans. Each header is clickable for sorting — show sort arrow icon (CaretUpIcon/CaretDownIcon from Phosphor) next to the active sort column.
    - Cell formatting by column.type:
      - `string`: render as-is, RTL text alignment
      - `number`: format with `toLocaleString('he-IL')`, left-align (numbers in RTL)
      - `date`: format with `new Date(val).toLocaleDateString('he-IL')`
      - `percentage`: format as `val.toFixed(1)%`
      - `currency`: format with `toLocaleString('he-IL', { style: 'currency', currency: 'ILS' })`
    - Empty state: if rows is empty, show a centered message "אין נתונים להצגה" with an EmptyState-style div.
    - Striped rows: alternate bg-muted/50 for readability.
    - Horizontal scroll wrapper for tables wider than viewport.

    **Summary section:**
    Below the table, render the summary.items as a row of small stat cards. Each item: `label` on top, `value` formatted by its `type` below. Use a flex row with gap, wrapped in a subtle bg-muted/30 rounded container.

    **Pagination:**
    Use the existing `Pagination` component from `../ui/Pagination`. Pass `currentPage`, `totalPages`, `totalItems`, `itemsPerPage`, `onPageChange`, `onItemsPerPageChange={onLimitChange}`, `entityLabel="רשומות"`.

    **Responsive:**
    - On mobile: horizontal scroll for table, summary items stack vertically.
    - Table min-width ensures readability.
  </action>
