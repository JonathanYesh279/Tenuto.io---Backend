  <name>Task 1: Create Room Utilization generator (DEPT-03)</name>
  <files>api/reports/generators/room-utilization.generator.js</files>
  <action>
Create `room-utilization.generator.js` following the established generator plugin convention.

**Generator metadata:**
- id: 'room-utilization'
- name: 'ניצולת חדרים' (Room Utilization)
- description: 'אחוז תפוסה, שעות שיא וזמינות לכל חדר'
- category: 'schedule'
- icon: 'Door'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - day: { type: 'string', required: false } -- filter to specific day (0-5 numeric)
- exports: ['excel', 'pdf']

**Columns:**
- room (string, sortable) -- 'חדר'
- totalActivities (number, sortable) -- 'פעילויות'
- occupiedMinutes (number, sortable) -- 'דקות תפוסות'
- occupancyPercent (percentage, sortable) -- 'אחוז תפוסה'
- peakHour (string) -- 'שעת שיא'
- availableSlots (number, sortable) -- 'משבצות פנויות'
- conflictCount (number, sortable) -- 'התנגשויות'

**generate(params, scope, { services }) logic:**

DO NOT import or call roomScheduleService -- query the data sources directly within the generator to avoid circular dependencies and to keep generators self-contained.

1. If scope.type === 'own', return emptyResult (room utilization not meaningful per-teacher).
2. Define operating hours: 08:00-20:00 (720 minutes per day). Use `VALID_DAYS` from constants.js for the 6-day week (Sun-Fri).
3. Determine which days to analyze: if params.day is specified, use just that day (0-5). Otherwise, analyze all 6 days (0-5).
4. For each day, collect all activities from three sources in parallel:
   a. **TimeBlocks:** Query teacher collection with aggregation pipeline (same approach as room-schedule.service.js `getTimeBlockActivities`):
      - Match `{ tenantId: scope.tenantId, isActive: true, 'teaching.timeBlocks': { $elemMatch: { day: hebrewDay, isActive: { $ne: false } } } }`
      - Unwind timeBlocks, filter by day
      - Extract: room (location), startTime, endTime
   b. **Rehearsals:** Query rehearsal collection `{ tenantId: scope.tenantId, dayOfWeek: day }` (add schoolYearId if params.schoolYearId). Extract: room (location), startTime, endTime.
   c. **Theory lessons:** Query theory_lesson collection `{ tenantId: scope.tenantId, dayOfWeek: day, isActive: true }` (add schoolYearId if params.schoolYearId). Extract: room (location), startTime, endTime.

5. Aggregate all activities across all analyzed days. Group by room name.
6. For each room, compute:
   - totalActivities: count of activities in this room
   - occupiedMinutes: sum of (endTime - startTime) in minutes for each activity, using a helper `timeToMinutes(time)` that converts "HH:MM" to minutes. Import `timeToMinutes` from `../../utils/timeUtils.js` (same utility used by room-schedule.service.js).
   - occupancyPercent: (occupiedMinutes / (720 * numberOfDaysAnalyzed)) * 100, rounded to 1 decimal
   - peakHour: find the hour (08, 09, ..., 19) with most overlapping activities. Build an hour-count array, find max. Display as "HH:00".
   - availableSlots: count 30-minute slots within 08:00-20:00 that have no activity overlap across analyzed days. For simplicity, compute as: (720 * numberOfDaysAnalyzed - occupiedMinutes) / 30, floored.
   - conflictCount: count activities that overlap with another activity in the same room on the same day (using `doTimesOverlap` from `../../utils/timeUtils.js`).

7. Skip activities with empty/null room (unassigned) -- don't include in report.
8. Sort rows by room name (Hebrew locale).

9. Summary items:
   - 'סה"כ חדרים' -- number of distinct rooms
   - 'סה"כ פעילויות' -- total activities across all rooms
   - 'ממוצע תפוסה' -- average occupancyPercent across rooms (percentage)
   - 'חדר עמוס ביותר' -- room name with highest occupancyPercent (string)
   - 'סה"כ התנגשויות' -- sum of conflictCount across all rooms

10. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Import `VALID_DAYS` from `../../../config/constants.js`. Import `timeToMinutes`, `doTimesOverlap` from `../../../utils/timeUtils.js`. Query collections directly via `services.getCollection()`.
  </action>
  <verify>
Call `GET /api/reports/registry` -- should include 'room-utilization'. Call `GET /api/reports/room-utilization` with auth -- should return valid response with per-room occupancy data. Call with `?day=0` -- should filter to Sunday only.
  </verify>
  <done>Room Utilization generator returns per-room occupancy percentage, peak hours, available slots, and conflict counts across all teaching days.</done>

<task type="auto">
  <name>Task 2: Create Teacher Schedule Density generator (DEPT-04)</name>
  <files>api/reports/generators/teacher-schedule-density.generator.js</files>
  <action>
Create `teacher-schedule-density.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'teacher-schedule-density'
- name: 'צפיפות מערכת מורים' (Teacher Schedule Density)
- description: 'כיסוי משבצות זמן וניתוח פערים במערכת מורים'
- category: 'schedule'
- icon: 'CalendarBlank'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - department: { type: 'string', required: false }
- exports: ['excel', 'pdf']

**Columns:**
- teacherName (string, sortable) -- 'שם מורה'
- totalBlocks (number, sortable) -- 'משבצות זמן'
- totalBlockMinutes (number, sortable) -- 'דקות מתוכננות'
- assignedLessonCount (number, sortable) -- 'שיעורים משובצים'
- assignedMinutes (number, sortable) -- 'דקות משובצות'
- utilizationPercent (percentage, sortable) -- 'ניצולת'
- activeDays (number, sortable) -- 'ימים פעילים'
- gapCount (number, sortable) -- 'פערים'
- longestGapMinutes (number, sortable) -- 'פער ארוך ביותר (דקות)'

**generate(params, scope, { services }) logic:**
1. Build teacher filter: `{ tenantId: scope.tenantId, isActive: true, 'teaching.timeBlocks.0': { $exists: true } }` (only teachers with at least one time block).
2. Department filter: if params.department, use `getInstrumentsByDepartment(params.department)`, add `{ 'personalInfo.instrument': { $in: instruments } }`.
3. Scope filtering:
   - scope.type === 'department': intersect instrument filter with scope departments.
   - scope.type === 'own': add `{ _id: services.toObjectId(scope.teacherId) }`.
4. Query teacher collection with projection: `{ personalInfo: 1, 'teaching.timeBlocks': 1 }`.
5. For each teacher, analyze their timeBlocks array:
   - totalBlocks: count of active time blocks (isActive !== false)
   - totalBlockMinutes: sum of (endTime - startTime) for each active block
   - assignedLessonCount: sum of active assignedLessons across all active blocks (lesson.isActive !== false)
   - assignedMinutes: sum of lesson durations (lesson.duration) across all active blocks. If lesson.duration not set, compute from lessonStartTime/lessonEndTime.
   - utilizationPercent: (assignedMinutes / totalBlockMinutes) * 100, rounded to 1 decimal. 0 if no blocks.
   - activeDays: count of distinct days (block.day) across active blocks
   - Gap analysis: for each day, sort blocks by startTime, compute gaps between consecutive blocks. A gap = time between block[i].endTime and block[i+1].startTime. Count gaps > 0 minutes.
   - gapCount: total gaps across all days
   - longestGapMinutes: max gap duration across all days. 0 if no gaps.

6. Map to rows with teacherName = `${teacher.personalInfo?.lastName || ''} ${teacher.personalInfo?.firstName || ''}`.trim().
7. Sort rows by teacherName.

8. Summary items:
   - 'סה"כ מורים עם מערכת' -- total teachers with time blocks
   - 'ממוצע ניצולת' -- average utilizationPercent (percentage)
   - 'ממוצע ימים פעילים' -- average activeDays (number)
   - 'סה"כ פערים' -- sum of gapCount across all teachers
   - 'מורה עם הכי הרבה פערים' -- teacher name with highest gapCount (string)

9. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Import `getInstrumentsByDepartment` from `../../../config/constants.js`. Import `timeToMinutes` from `../../../utils/timeUtils.js` for gap computation.
  </action>

Call `GET /api/reports/registry` -- should include 'teacher-schedule-density'. Call `GET /api/reports/teacher-schedule-density` with auth -- should return per-teacher schedule data with gap counts.

  <files>api/reports/generators/schedule-overview.generator.js</files>
  <action>
Create `schedule-overview.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'schedule-overview'
- name: 'מערכת תזמורות ותאוריה' (Orchestra/Theory Schedule)
- description: 'לוח זמנים שבועי של הרכבים ושיעורי תאוריה עם שיבוץ מורים וחדרים'
- category: 'schedule'
- icon: 'Calendar'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - day: { type: 'string', required: false } -- filter to specific day (0-5 numeric)
  - activityType: { type: 'string', required: false, default: 'all', allowed: ['all', 'rehearsal', 'theory'] }
- exports: ['excel', 'pdf']

**Columns:**
- dayName (string, sortable) -- 'יום'
- startTime (string, sortable) -- 'שעת התחלה'
- endTime (string) -- 'שעת סיום'
- activityName (string, sortable) -- 'פעילות'
- activityType (string, sortable) -- 'סוג'
- teacherName (string, sortable) -- 'מורה/מנצח'
- room (string, sortable) -- 'חדר'
- memberCount (number, sortable) -- 'משתתפים'

**generate(params, scope, { services }) logic:**
1. If scope.type === 'own', return emptyResult (schedule overview not meaningful for individual teacher).
2. Determine which days to query: if params.day specified, just that day. Otherwise, all 6 days (0-5).
3. Import `VALID_DAYS` from constants.js for Hebrew day names.
4. For each day, query in parallel:
   a. **Rehearsals:** Query rehearsal collection `{ tenantId: scope.tenantId, dayOfWeek: day }` (add schoolYearId if present). Project: `{ groupId, dayOfWeek, startTime, endTime, location, type }`. Deduplicate by groupId+dayOfWeek+startTime+endTime (same approach as room-schedule.service.js, but simpler -- just use distinct combination).
   b. **Theory lessons:** Query theory_lesson collection `{ tenantId: scope.tenantId, dayOfWeek: day, isActive: true }` (add schoolYearId). Project: `{ category, teacherId, dayOfWeek, startTime, endTime, location }`. Deduplicate by category+teacherId+dayOfWeek+startTime.

5. Batch-resolve names:
   - Collect all orchestraIds (groupId from rehearsals). Query orchestra collection `{ _id: { $in: orchestraObjectIds } }` to get name, conductorId, and memberIds.length for memberCount.
   - Collect all teacherIds (conductorIds from orchestras + teacherId from theory lessons). Query teacher collection to get names.

6. Build rows:
   - For rehearsals: dayName = VALID_DAYS[dayOfWeek], activityName = orchestra name, activityType = 'חזרה', teacherName = conductor name from lookup, room = location, memberCount = orchestra.memberIds.length.
   - For theory lessons: dayName = VALID_DAYS[dayOfWeek], activityName = category (e.g., 'תורת המוסיקה'), activityType = 'תאוריה', teacherName from lookup, room = location, memberCount = 0 (theory lessons don't have member count in collection -- set to '-' or 0).

7. If params.activityType === 'rehearsal', filter to rehearsals only. If 'theory', filter to theory only.

8. Sort rows by day (numeric), then startTime.

9. Summary items:
   - 'סה"כ פעילויות' -- total row count
   - 'חזרות' -- count of rehearsal rows
   - 'שיעורי תאוריה' -- count of theory rows
   - 'חדרים בשימוש' -- distinct room count
   - 'מורים/מנצחים' -- distinct teacher count

10. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Use `ObjectId` from 'mongodb' for ID conversions. Import `VALID_DAYS` from `../../../config/constants.js`. Use the same deduplication approach as room-schedule.service.js for rehearsals (group by groupId+day+startTime+endTime+location).
  </action>
