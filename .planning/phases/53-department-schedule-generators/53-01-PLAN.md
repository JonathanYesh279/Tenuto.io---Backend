  <name>Task 1: Create Department Overview generator (DEPT-01)</name>
  <files>api/reports/generators/department-overview.generator.js</files>
  <action>
Create `department-overview.generator.js` following the established generator plugin convention.

**Generator metadata:**
- id: 'department-overview'
- name: 'סקירת מחלקות' (Department Overview)
- description: 'נתוני תלמידים, מורים, שעות ותזמורות לכל מחלקה'
- category: 'department'
- icon: 'ChartBar'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params:
  - department: { type: 'string', required: false } -- filter to specific department
- exports: ['excel', 'pdf']

**Columns:**
- department (string, sortable) -- 'מחלקה'
- studentCount (number, sortable) -- 'תלמידים'
- teacherCount (number, sortable) -- 'מורים'
- totalWeeklyHours (number, sortable) -- 'שעות שבועיות'
- avgHoursPerTeacher (number, sortable) -- 'ממוצע שעות למורה'
- orchestraCount (number, sortable) -- 'תזמורות/הרכבים'

**generate(params, scope, { services }) logic:**
1. If scope.type === 'own', return emptyResult (department overview not meaningful for individual teacher).
2. Get all 9 departments from INSTRUMENT_DEPARTMENTS imported from `../../../config/constants.js`.
3. If params.department specified, filter to just that department.
4. If scope.type === 'department', filter to only scope.departmentIds.
5. For each department, collect metrics in parallel:
   a. **Students:** Query student collection `{ tenantId: scope.tenantId, isActive: true, 'academicInfo.instrumentProgress.instrumentName': { $in: instrumentsForDept } }`. Use `getInstrumentsByDepartment(dept)` from constants.js to get the instruments for that department. Count documents.
   b. **Teachers:** Query teacher collection `{ tenantId: scope.tenantId, isActive: true, 'personalInfo.instrument': { $in: instrumentsForDept } }`. Count documents. (A teacher's primary instrument is in `personalInfo.instrument`.)
   c. **Hours:** Query hours_summary collection `{ tenantId: scope.tenantId }` (add schoolYearId if params.schoolYearId). Filter in-memory: for each hours_summary doc, lookup the teacher's instrument, check if it maps to current department. Sum `totals.totalWeeklyHours`.
   d. **Orchestras:** This is harder to map to departments directly. Skip orchestra count per department -- instead show total orchestras across institution for reference in summary only.

   **Simpler approach for hours and teachers:**
   - Fetch ALL teachers for tenant `{ tenantId, isActive: true }`, project `{ personalInfo.instrument: 1 }`. Build a map: teacherId -> department (via `getInstrumentDepartment(teacher.personalInfo.instrument)`).
   - Fetch ALL hours_summary docs for tenant (with schoolYearId filter if present). Group by teacherId, lookup department from teacher map, aggregate hours per department.
   - Count teachers per department from the teacher map.
   - This avoids N queries (one per department).

6. For student counts, a single query approach: fetch all active students with instrumentProgress, then group by primary instrument's department in-memory.

7. Build one row per department with the aggregated metrics. Calculate avgHoursPerTeacher = totalWeeklyHours / teacherCount (0 if no teachers).

8. Sort rows by department name.

9. Summary items:
   - 'סה"כ מחלקות' -- number of departments with data
   - 'סה"כ תלמידים' -- sum of all studentCount
   - 'סה"כ מורים' -- sum of all teacherCount
   - 'סה"כ שעות שבועיות' -- sum of all totalWeeklyHours
   - 'ממוצע שעות למחלקה' -- totalWeeklyHours / departments with data

10. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Import `INSTRUMENT_DEPARTMENTS`, `getInstrumentDepartment`, `getInstrumentsByDepartment` from `../../../config/constants.js`.
  </action>
  <verify>
Start the server and call: `curl -s http://localhost:PORT/api/reports/registry | jq '.[] | select(.id == "department-overview")'` -- should return the generator metadata. Then call `GET /api/reports/department-overview` with auth -- should return valid { metadata, columns, rows, summary } response with one row per department.
  </verify>
  <done>Department Overview generator is auto-discovered by registry, returns one row per department with student/teacher/hours counts, and summary with totals across all departments.</done>

  <name>Task 2: Create Department Comparison generator (DEPT-02)</name>
  <files>api/reports/generators/department-comparison.generator.js</files>
  <action>
Create `department-comparison.generator.js` following the generator plugin convention.

**Generator metadata:**
- id: 'department-comparison'
- name: 'השוואת מחלקות' (Department Comparison)
- description: 'השוואה צד-אל-צד של מדדים בין כל 9 המחלקות'
- category: 'department'
- icon: 'Table'
- roles: ['מנהל', 'סגן מנהל', 'מזכירות']
- params: {} (no extra params -- always shows all 9 departments)
- exports: ['excel', 'pdf']

**Columns:**
- metric (string, sortable) -- 'מדד'
- Then one column per department (9 columns). Since the contract requires static columns, use a transposed approach:
  Instead, use a row-per-metric layout (like year-over-year-comparison.generator.js):
- metric (string, sortable) -- 'מדד'
- For each of the 9 departments, a separate column would violate the static column contract. Instead, use the same approach as the overview but with additional computed metrics:

**Better approach -- row-per-department with richer metrics:**
- department (string, sortable) -- 'מחלקה'
- studentCount (number, sortable) -- 'תלמידים'
- teacherCount (number, sortable) -- 'מורים'
- studentTeacherRatio (number, sortable) -- 'יחס תלמיד/מורה'
- totalWeeklyHours (number, sortable) -- 'שעות שבועיות'
- avgHoursPerTeacher (number, sortable) -- 'ממוצע שעות למורה'
- avgHoursPerStudent (number, sortable) -- 'ממוצע שעות לתלמיד'
- percentOfTotalHours (percentage, sortable) -- 'אחוז משעות כוללות'
- percentOfTotalStudents (percentage, sortable) -- 'אחוז מתלמידים'

**generate(params, scope, { services }) logic:**
1. If scope.type === 'own', return emptyResult.
2. Use the same data-fetching strategy as department-overview: fetch all teachers, all hours_summary docs, all active students in single bulk queries. Group by department.
3. If scope.type === 'department', filter to only scope.departmentIds.
4. Compute per-department:
   - studentCount, teacherCount, totalWeeklyHours (same as overview)
   - studentTeacherRatio: studentCount / teacherCount (round to 2 decimals, 0 if no teachers)
   - avgHoursPerTeacher: totalWeeklyHours / teacherCount (round to 2 decimals)
   - avgHoursPerStudent: totalWeeklyHours / studentCount (round to 2 decimals, 0 if no students)
5. Calculate totals across all departments for percentages:
   - percentOfTotalHours: (deptHours / totalHoursAllDepts) * 100, rounded to 1 decimal
   - percentOfTotalStudents: (deptStudents / totalStudentsAllDepts) * 100, rounded to 1 decimal
6. Always show all 9 departments (even if some have 0 counts), so admins see the full picture.
7. Sort rows by department name (Hebrew locale).

8. Summary items:
   - 'סה"כ מחלקות פעילות' -- departments with at least 1 student or teacher
   - 'מחלקה גדולה ביותר (תלמידים)' -- department name with most students
   - 'מחלקה גדולה ביותר (שעות)' -- department name with most hours
   - 'יחס תלמיד/מורה ממוצע' -- overall studentCount / teacherCount

9. Return `{ columns: this.columns, rows, summary: { items } }`.

**Important:** Reuse the same bulk-fetch-and-group pattern from department-overview. Import constants from `../../../config/constants.js`.
